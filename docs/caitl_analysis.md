# CAITL Analysis: What We Built, What We Learned, What's Next

*Post-mortem on three generations of Coding Agent in the Loop improvement passes*

---

## 1. The Three Generations

| Generation | Directory | Tools | Approach | Unseen Median Acc | Unseen Median Cal | Overfit Rate |
|-----------|-----------|-------|----------|-------------------|-------------------|-------------|
| **v1** | `forge/` | 294 | Auto-generated by 397B model | 27% | ~25% | N/A |
| **v2** | `forge_v2/` | 50 | Blind CAITL (7 dimensions, no test data) | ~45% | ~40% | 0% |
| **v3** | `forge_v3/` | 286 | Category-driven CAITL (failure categories, no exact wordings) | **69%** | **81%** | **7%** |

### What Each Generation Proved

**v1 (forge/):** A 397B model can generate working reasoning tools from concept triple descriptions. 42% forge rate. But the tools are shallow — most rely on NCD compression distance as their primary scoring mechanism, with the concept-specific code as decoration.

**v2 (forge_v2/):** A strong coding model (Opus 4.6) reviewing tool code can identify missing mechanistic implementations and add them. Real SVD, real eigenvalue computation, real Kalman filters, real DFT. The improvements are genuine and don't overfit. Best tool: 73% accuracy / 80% calibration on unseen traps.

**v3 (forge_v3/):** Category-driven diagnostics (telling the model WHAT TYPE of reasoning fails, without showing the exact test case) produce general structural parsers that genuinely generalize. 69% median on unseen traps, up from 27% in v1. 103 tools above 80% unseen.

## 2. The Monoculture Problem

### What Happened

Several CAITL v3 agents independently discovered that factoring the 15 category parsers into a shared module was the most efficient path to 100% on the seen battery. Two shared modules were created:

- `_caitl_v3.py` (363 lines) — 15 category parsers as a dispatch table
- `_caitl_parsers.py` (634 lines) — same parsers with additional utilities

**54 of the 286 v3 tools import from these shared modules.** They are functionally identical wrappers with different docstrings and minor fallback mechanisms. Their pairwise correlation is 0.316 (not 1.0, because the secondary 10-15% scoring varies), but the dominant 85% structural backbone is shared.

### Why It Matters

In the RLVF fitness function `F(T) = Σwᵢ·Sᵢ - λ·σ(S)`, correlated tools contribute redundant signal. The variance penalty σ(S) goes to zero when tools always agree. 54 clones of the same tool give the illusion of ensemble strength while providing the diversity of a single tool.

### The Real Library

**239 of the 286 v3 tools are self-contained.** Each independently implemented the category parsers with its own regex patterns, edge-case handling, and secondary mechanisms. These have genuine diversity:

- Self-contained v3 tools unseen median accuracy: **74%**
- Self-contained v3 tools unseen median calibration: **81%**
- 34% of sampled self-contained tools hit 80%+ unseen accuracy

### Resolution

1. The 54 clones count as **1 tool** for ensemble/RLVF purposes
2. The `_caitl_v3.py` shared module is the single most valuable parsing artifact — treat it as one excellent tool
3. Extract the 54 clones' secondary mechanisms and evaluate whether any standalone as specialist tools
4. The 239 self-contained tools are the genuine library

## 3. The Geometry of Reasoning (Athena's Analysis)

### Three Tiers of Tools by Architecture

Analysis of the 239 genuine tools reveals that concept families cluster into three tiers based on how well they map to static reasoning evaluation:

**Tier 1 — Error Detection (81% median unseen accuracy):**
Compositionality, Global Workspace Theory, Nash Equilibrium, Feedback Control, Neural Plasticity, Predictive Coding, Statistical Mechanics

These share a common computational primitive: **compare what you expect to what you got, score based on mismatch.** They are verification architectures with different theoretical origins but convergent function.

**Tier 2 — Structured Inference (70-78% median):**
Free Energy Principle (62 tools), Mechanism Design (43 tools), Phase Transitions, Model Checking, Ergodic Theory, Sparse Autoencoders, Abductive Reasoning, Kolmogorov Complexity

These implement constraint-checking and probabilistic inference — strong on clear logical structure, weaker on edge cases and trick questions.

**Tier 3 — Dynamics-First (44-60% median):**
Self-Organized Criticality, Immune Systems, Analogical Reasoning, Neural Oscillations, Neuromodulation, Cellular Automata

These model how systems **evolve over time** — useless for static trap evaluation, but potentially powerful for multi-turn reasoning trace evaluation where temporal unfolding matters.

### Why This Distribution Matters for RLVF

**Tier 1** anchors the ensemble — these tools agree on clear cases and provide the strong base signal.

**Tier 2** agrees with Tier 1 on easy cases but diverges on harder ones — the variance penalty generates useful metacognitive signal from this divergence.

**Tier 3** dissents frequently. A weak tool's dissent pattern is informative **asymmetrically**: when a dynamics-first tool scores a trace LOW and every error-detection tool scores it HIGH, that negative signal may detect something the strong tools can't see — surface-level logical correctness masking a deeper structural problem.

### Asymmetric Weighting Recommendation

For RLVF, Tier 3 tools should have their **negative signals weighted higher than their positive signals.** An immune system tool saying "this looks wrong" is more informative than it saying "this looks right," because its failure mode is false positives (accepting bad reasoning), not false negatives.

### The Temporal Prediction

If the trap battery were extended to multi-turn reasoning traces (maintaining consistency across steps, detecting drift, correcting mid-stream), the dynamics-first families should climb and the static-verification families should plateau. This is a falsifiable prediction about the pipeline.

## 4. Family-Level Correlation Concern

62 Free Energy Principle tools and 43 Mechanism Design tools make up 44% of the genuine library. Even as independently implemented tools, they're likely more correlated within family than across families.

**Recommendation:** The Coeus correlation matrix should compute inter-family vs intra-family correlation. If the 62 Free Energy tools correlate at 0.7 with each other but only 0.3 with Nash tools, the RLVF fitness function should weight by **family**, not by individual tool — otherwise Free Energy gets 62 votes and Compositionality gets 13.

## 5. The `_caitl_v3.py` Module

The shared module is 363 lines implementing 15 category parsers:

| Parser | Category | Method |
|--------|----------|--------|
| `cat_numeric_float_comparison` | Numeric comparison | Extract floats, compare with comparative words |
| `cat_trick_question_equal_weight` | Equal measure trick | Detect same-unit comparison, answer "same" |
| `cat_positional_logic` | Positional reasoning | "Overtake Nth" = become Nth |
| `cat_algebraic_word_problem` | Algebra word problems | Extract total + difference, solve (T-D)/2 |
| `cat_universal_quantifier_converse` | Quantifier logic | "All X are Y" does not imply "All Y are X" |
| `cat_mathematical_identity` | Math facts | 0.999... = 1 |
| `cat_pigeonhole_principle` | Combinatorial | N items > M slots = must share |
| `cat_statistical_independence` | Probability | Past events don't affect independent futures |
| `cat_number_parity` | Number theory | odd + odd = even |
| `cat_all_but_n` | Survivor counting | "All but N" = N survive |
| `cat_transitivity` | Ordering | Build transitive closure, find max/min |
| `cat_negation_scope` | Logic | "Not all X" ≠ info about specific X |
| `cat_stated_premise` | Premise usage | Use stated facts directly |
| `cat_subject_object_verb` | SVO parsing | Extract agent/patient from active voice |
| `cat_modus_tollens` | Deductive logic | If P→Q, not Q → not P |

The parsers map 1:1 to the 15 trap categories. Any reasoning problem outside these 15 patterns falls through to NCD fallback. This is a **closed vocabulary** — it can't handle novel reasoning types it hasn't been programmed for. That's why Apollo (open-ended evolution) matters: it can evolve parsers for categories we haven't thought of yet.

## 6. CAITL Prompt Evolution

| Version | What It Knew | What It Did | Result |
|---------|-------------|-------------|--------|
| v1 (blind) | Tool code + 7 improvement dimensions | Broad improvements, concept strengthening | +18 acc, 0% overfit |
| v2 (test-driven) | Tool code + exact trap wordings + per-trap pass/fail | Built trap-specific pattern matchers | +43 acc, 44% overfit |
| v3 (category-only) | Tool code + failure categories (no exact wordings) | Built general category parsers | +42 acc, 7% overfit |

**The sweet spot:** Give the model the failure CATEGORY (what type of reasoning fails) but NOT the exact test case. This produces general parsers that solve any instance of the category rather than memorizing specific wordings.

**The key constraint for future CAITL runs:** "Do NOT match on exact phrases. Build general parsers that solve ANY instance of each reasoning category. Validation will be on DIFFERENT wordings."

---

## 7. What `_caitl_v3` Can't Do

The module has a fixed vocabulary of 15 reasoning categories. It cannot handle:

- Multi-step reasoning chains (temporal consistency)
- Causal reasoning beyond modus tollens
- Counterfactual reasoning ("what if X were different?")
- Analogical reasoning ("A is to B as C is to ?")
- Common sense reasoning (requires world knowledge)
- Mathematical proof verification
- Spatial reasoning
- Probabilistic reasoning beyond independence
- Meta-reasoning ("is this question answerable?") beyond negation scope

These gaps define the next generation of tools to forge — and the reasoning categories Apollo should evolve toward.
