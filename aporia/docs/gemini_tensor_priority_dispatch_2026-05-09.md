Each report must include:

1. **Problem Statement** — exact formulation; current best bounds.
2. **Status & Bounds** — what's known unconditionally; what's known conditionally; key timestamps.
3. **Literature** — 2024-2025 references prioritized; key 2010s+ references; named tools (TensorLy, Macaulay2, Bertini, ITensor, etc.).
4. **Attack Vectors Active in the Literature** — name the paradigms (P01-P31) where they apply; flag any new attack patterns not yet in our taxonomy.
5. **Substrate Encoding** — propose how this problem would encode in current substrate primitives; flag the gap if any.
6. **Calibration Anchor Notes** — what counts as substrate-grade vs textbook-trivial response on this problem; what fabrications a Learner might produce; what canonical attributions are at risk.
7. **Cross-References** — to other entries in `tensor_open_problems_v1.md`; to existing reports in `aporia/docs/deep_research_batch*/`; to substrate-tester capability-gap tickets.

Each report MUST cite at least 2 of these patterns by name (per recurring discipline):
PATTERN_PRIME_GRAVITATIONAL_OVERFIT, PATTERN_CONDUCTOR_CONFOUND, PATTERN_BASE_RATE_NEGLECT, PATTERN_VRAM_TRUNCATION_ARTIFACT, PATTERN_RANK_PARITY_LEAK.
