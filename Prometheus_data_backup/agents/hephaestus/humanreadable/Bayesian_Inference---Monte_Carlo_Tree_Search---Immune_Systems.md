**Algorithm (Bayesian‑MCTS‑Immune Scorer)**  
1. **Parsing & Proposition Extraction** – Using only `re` we extract clause‑level propositions from the prompt and each candidate answer. Each proposition is encoded as a feature vector `x ∈ {0,1}^k` where dimensions capture: presence of negation, comparative operator (`>`, `<`, `=`), conditional connective (`if…then`), causal cue (`because`, `leads to`), numeric token (scaled to [0,1]), ordering relation (`before`, `after`), and quantifier (`all`, `some`, `none`). The vector is stored as a NumPy array.  
2. **Belief Tree (MCTS)** – Each node corresponds to a set of propositions (a partial interpretation). Nodes store:  
   - `prior`: Beta(α,β) parameters representing initial belief in the node’s truth (α=1,β=1 uniform).  
   - `visit_count`, `value_sum`.  
   - Children are generated by **expansion**: flipping a negation, swapping a comparative direction, or alternative numeric binding (e.g., “5 > 3” vs “3 < 5”).  
   - **Selection** uses UCB1: `value = value_sum/visit_count + c*sqrt(log(parent_visits)/visit_count)`.  
   - **Simulation** rolls out a random completion of the tree by sampling truth values for leaf propositions from their current Beta posteriors; the likelihood of the candidate answer given the sampled world is computed as the product of Bernoulli probabilities (conjugate update).  
   - **Backpropagation** updates `visit_count` and adds the simulated likelihood to `value_sum`.  
3. **Immune‑Inspired Clonal Selection** – After a fixed number of MCTS iterations, we have a population of “antibodies”: each antibody is a complete world assignment (a leaf node) with affinity = posterior probability of the candidate answer under that world.  
   - **Selection**: keep top‑`M` antibodies by affinity.  
   - **Cloning**: duplicate each selected antibody proportionally to its affinity.  
   - **Mutation**: with probability `p_mut` randomly flip a proposition feature (e.g., toggle negation, perturb numeric value).  
   - **Evaluation**: recompute affinity for mutants using the same Bayesian likelihood.  
   - **Memory**: the highest‑affinity antibody seen so far is stored and its affinity contributes a prior boost to similar future nodes (via increased α).  
4. **Scoring** – The final score for a candidate answer is the normalized affinity of the memory antibody (or the mean of the top‑`M` affinities), yielding a value in `[0,1]`. Higher scores indicate stronger logical support from the prompt.
