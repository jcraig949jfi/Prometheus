**Algorithm**  
We build a hierarchical, attention‑guided clonal‑selection scorer that works entirely with NumPy and the Python stdlib.

1. **Proposition extraction** – Using a handful of regex patterns we scan the prompt and each candidate answer for:  
   - Negations (`not`, `no`)  
   - Comparatives (`more than`, `less than`, `≥`, `≤`)  
   - Conditionals (`if … then`, `unless`)  
   - Causal markers (`because`, `leads to`, `results in`)  
   - Ordering/temporal terms (`before`, `after`, `while`)  
   - Numeric values (integers, floats, percentages)  
   Each match yields a proposition dict: `{type, polarity, args, value}` where `args` are the extracted spans and `value` holds any numeric constant.

2. **Feature vectors** – For every proposition we create a fixed‑length NumPy vector: one‑hot encoding of `type` (negation, comparative, conditional, causal, ordering, numeric), a normalized scalar for any extracted number, and a binary flag for polarity. All vectors are stacked into matrix **F** ∈ ℝⁿˣᵈ.

3. **Attention weighting** – Compute similarity **S** = F·Fᵀ, apply row‑wise softmax to obtain attention matrix **A** ∈ ℝⁿˣⁿ, where Aᵢⱼ reflects how relevant proposition *j* is to *i* given their structural features.

4. **Clonal‑selection population** – Initialise a population of *P* reasoning chains. Each chain is a list of proposition indices generated by a weighted random walk on **A** (higher attention → higher transition probability).  

5. **Scoring a chain** – For a chain we evaluate:  
   - Logical constraints: propagate negations, apply modus ponens on conditionals, enforce transitivity on ordering, check consistency of comparatives using the extracted numeric values.  
   - Numeric satisfaction: compute violations of comparative/threshold constraints (e.g., if a proposition states “X > 5” and the extracted value for X is 3, add penalty |3‑5|).  
   The chain score = –(logical penalty + numeric violation). Higher is better.

6. **Renormalization (coarse‑graining)** – After *G* generations:  
   - Identify pairs (i,j) with Aᵢⱼ > τ (threshold). Merge each pair into a super‑node, summing their feature vectors and recomputing **F** and **A** for the reduced graph.  
   - Propagate scores upward: a super‑node inherits the best chain score of its constituents.  
   - Iterate until the change in best score between successive coarse‑graining steps < ε (fixed point).  

The final score for a candidate answer is the best chain score obtained at the coarsest stable scale.
