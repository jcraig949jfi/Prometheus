## Nous Analysis

**Algorithm: Typed Dialectical Abstract Interpreter (TDAI)**  

1. **Parsing & Typing** – The input sentence is tokenized with a regex‑based shallow parser that extracts predicates of the form `Rel(arg1, arg2, …)`. Each predicate is assigned a simple type from a finite lattice:  
   - `Bool` for propositions (e.g., `IsRainy`)  
   - `Int` or `Real` for numeric terms (e.g., `Age`)  
   - `Ord` for ordered relations (`>`, `<`, `=`)  
   - `Causal` for cause‑effect patterns (`because`, `leads to`).  
   The parser builds a typed abstract syntax tree (AST) where each node stores its type and a list of child nodes.

2. **Dialectical Expansion** – For every extracted proposition `p` a thesis node `T(p)` is created. An antithesis node `A(p)` is generated by applying syntactic negation (`not`) or by swapping the arguments of an ordered/comparative predicate (e.g., `x > y` → `y > x`). A synthesis node `S(p)` is initially undefined.

3. **Constraint Propagation (Abstract Interpretation)** –  
   - Each node carries an abstract value in a lattice:  
     * `Bool`: `{⊥, False, True, ⊤}` (⊥ = false, ⊤ = true, ⊤ = over‑approx).  
     * `Int/Real`: interval `[l, u]` stored as a length‑2 numpy array.  
     * `Ord`: direction sign (`-1,0,+1`).  
     * `Causal`: confidence interval `[0,1]`.  
   - Initialization: thesis gets the concrete truth value derived from the prompt (e.g., if the prompt states “It is raining”, `T(IsRainy)` gets `[True,True]`); antithesis gets the opposite bound; synthesis starts as `⊥`.  
   - Propagation rules are simple logical constraints encoded as numpy operations:  
     * Modus ponens: if `T(P)` is `True` and `T(P→Q)` is `True` then update `T(Q)` with `⊔` (join) of its current value and `True`.  
     * Transitivity for `Ord`: `x > y` ∧ `y > z` ⇒ update interval for `x > z` using `np.maximum`.  
     * Contradiction detection: if a node’s interval becomes empty (`l>u` for numerics or `False ∧ True` for bools) mark a conflict.  
   - Iterate until a fixed point (no changes) or a max depth of 5 passes.

4. **Scoring** – For a candidate answer, repeat steps 1‑3 to obtain its synthesized abstract value `S_cand`. Compare it to the reference answer’s synthesized value `S_ref` using a distance metric:  
   - Bool: `0` if both same concrete value, `1` if one is `⊤` and the other concrete, `2` if opposite concretions.  
   - Numeric: normalized L1 distance of intervals: `(|l_cand-l_ref|+|u_cand-u_ref|)/(max_range)`.  
   - Overall score = `1 - (weighted sum of distances)`, clipped to `[0,1]`. Lower distance → higher score.

**Structural Features Parsed** – negations (`not`, `no`), comparatives (`>`, `<`, `>=`, `<=`), conditionals (`if … then …`), causal cues (`because`, `leads to`, `results in`), numeric constants, ordering chains, and simple quantifiers (`all`, `some`) via keyword triggers.

**Novelty** – The combination mirrors existing work (typed lambda calculi in proof assistants, abstract interpretation in static analysis, and dialectical refinement in argumentation systems) but fuses them into a single, lightweight, numpy‑based scoring loop. No prior public tool explicitly couples typed dialectical expansion with interval abstract interpretation for answer scoring, making the approach novel in this niche.
