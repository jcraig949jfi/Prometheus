## Nous Analysis

**Algorithm**  
The scorer builds a lightweight, trainable scoring function by treating each candidate answer as a noisy transmission of an underlying “correct” representation.  

1. **Feature extraction (structural parser)** – Using only regex and the Python `re` module, the prompt and each candidate answer are scanned for a fixed set of linguistic primitives:  
   * negation tokens (`not`, `no`, `never`)  
   * comparatives (`more`, `less`, `greater`, `fewer`, `-er`, `more … than`)  
   * conditionals (`if`, `unless`, `provided that`)  
   * numeric values (integers, decimals, fractions)  
   * causal cue verbs (`cause`, `lead to`, `result in`, `because`)  
   * ordering relations (`before`, `after`, `first`, `last`, `increasing`, `decreasing`)  
   Each primitive contributes a binary feature; the presence/absence of a token yields a 1/0 entry. The resulting feature vector **x** ∈ {0,1}^d is stored as a NumPy array.

2. **Error‑correcting encoding** – To make the scorer robust to superficial variations, **x** is encoded with a systematic binary linear code (e.g., a (n,k) Hamming or Reed‑Solomon‑derived binary code). The encoder **G** (k×n) produces a codeword **c = x·G** ∈ {0,1}^n. Redundancy bits allow the scorer to recover the underlying feature pattern even when up to t bits are flipped.

3. **Property‑based test generation** – Inspired by Hypothesis, a generator creates perturbed codewords **c'** by randomly flipping up to t bits (simulating noise) and, crucially, by applying *semantic* perturbations: swapping a comparative with its opposite, negating a causal cue, or inverting an ordering relation. Each **c'** is decoded (via syndrome decoding) to obtain a hypothesis feature vector **x'**.

4. **Neural Architecture Search (NAS) for scoring** – The search space consists of tiny feed‑forward networks with at most two hidden layers, each layer limited to 4–8 units, and weight sharing across layers (i.e., the same weight matrix is reused for both hidden layers). The network takes **x** (or **x'**) and outputs a scalar score s ∈ ℝ. The NAS algorithm enumerates all architectures in this micro‑space, trains each for a fixed few epochs on a synthetic dataset generated by the property‑based tester (positive examples: codewords with ≤t flips; negative examples: codewords with >t flips), and records validation accuracy. Weight sharing drastically reduces training cost, allowing exhaustive search with only NumPy operations (matrix multiplies, ReLU, simple SGD).

5. **Scoring logic** – After NAS selects the best architecture, its weights are fixed. To score a new candidate answer, the parser extracts **x**, encodes to **c**, and the network outputs s. The final score is the sigmoid‑scaled s, yielding a value in [0,1] that reflects how well the answer satisfies the extracted structural properties under noise‑robust conditions.
