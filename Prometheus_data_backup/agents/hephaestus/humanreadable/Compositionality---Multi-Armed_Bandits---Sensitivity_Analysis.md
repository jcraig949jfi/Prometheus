## Nous Analysis

**Algorithm**  
1. **Parsing layer (compositionality)** – Using regex‑based patterns we extract atomic propositions from the prompt and each candidate answer. Each proposition is encoded as a feature vector *f* ∈ ℝᵏ where dimensions correspond to: presence of negation, comparative operator, conditional antecedent/consequent, numeric value (normalized), causal verb, ordering relation (“greater‑than”, “before”), and quantifier scope. The whole answer is represented by the sum of its proposition vectors *F* = ∑ *fᵢ*.  
2. **Weight vector (bandit arm)** – A weight vector *w* ∈ ℝᵏ determines how much each feature contributes to the answer’s plausibility score. We treat each possible perturbation of *w* as an arm of a multi‑armed bandit. The baseline *w₀* is set to uniform weights; arms are generated by adding a small random δ ∈ [−ε,ε]ᵏ (ε=0.1).  
3. **Scoring & update** – For a given arm, the raw score is *s* = σ(w·F) where σ is the logistic sigmoid (implemented with numpy). The reward *r* is defined as the negative sensitivity of the score to weight perturbations:  
    *r* = −‖∇₍w₎ s‖₂ = −σ′(w·F)·‖F‖₂.  
   This rewards answers whose score is stable (low gradient) under small weight changes, i.e., robust to model misspecification.  
   After evaluating a candidate with arm a, we update its UCB value:  
    UCBₐ = meanₐ + √(2 ln N / nₐ),  
   where meanₐ is the average reward observed for arm a, N total evaluations, nₐ pulls of arm a. The arm with highest UCB is selected for the next candidate, implementing the explore‑exploit trade‑off.  
4. **Final selection** – After a fixed budget of evaluations (e.g., 30 pulls), the candidate with the highest average score across all arms is returned.
