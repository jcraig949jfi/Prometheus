**Algorithm**  
We build a directed proof‑graph G = (V,E) where each vertex v ∈ V represents a proposition extracted from the prompt or a candidate answer. Edges encode logical relations: implication (→), equivalence (↔), negation (¬), ordering (≤, ≥), and causal links (because, leads to). Every vertex stores a binary codeword c(v) ∈ {0,1}^L generated by a fixed linear error‑correcting code (e.g., an LDPC parity‑check matrix H). The reference codeword for a proposition is obtained by syndrome‑free encoding of its canonical form; the candidate’s proposition yields a possibly different codeword ĉ(v).  

1. **Parsing & encoding** – Regex patterns pull out propositions and annotate them with the logical operators above. Each proposition is transformed into a normalized string (lower‑cased, units stripped) and fed to the encoder to produce c(v).  
2. **Error measurement** – For each vertex compute the syndrome s(v)=H·ĉ(v) (mod 2). The Hamming weight w(v)=|s(v)| quantifies how far the candidate’s encoding deviates from the reference.  
3. **Load initialization** – Set the initial load ℓ(v)=w(v)·deg⁺(v) (out‑degree weighted error).  
4. **Self‑organized criticality dynamics** – Treat G as a sandpile: each vertex has a threshold θ=1 (can be tuned). While ∃v with ℓ(v)≥θ, topple v: ℓ(v)←ℓ(v)−θ·deg(v); for each u∈N(v) (in‑ and out‑neighbors) do ℓ(u)←ℓ(u)+θ/|N(v)|. Record the number of topplings T (avalanche size). The process terminates because total load is finite and each topple reduces the ℓ₁‑norm.  
5. **Scoring** – Define score S=1/(1+α·T) with α≈0.1 to damp large avalanches. Higher S indicates fewer logical inconsistencies, i.e., a better‑reasoned answer.
