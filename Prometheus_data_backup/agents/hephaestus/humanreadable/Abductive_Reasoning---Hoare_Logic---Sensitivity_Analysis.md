**Algorithm: Abductive‑Hoare Sensitivity Scorer (AHSS)**  

*Data structures*  
- **Clause graph**: each sentence → node; directed edges labeled with extracted logical relations (implication, equivalence, negation, ordering, causal). Stored as adjacency lists of tuples `(target, rel_type, weight)`.  
- **State vector** `s ∈ ℝ^m`: one dimension per distinct predicate (e.g., `X>Y`, `cause(A,B)`, `¬P`). Initialized from the prompt’s factual assertions (value = 1 if true, 0 if false, 0.5 if unknown).  
- **Hypothesis set** `H = {h₁,…,h_k}`: each hypothesis is a sparse binary vector `h ∈ {0,1}^m` representing a possible completion of missing predicates. Generated by abductive step (see below).  
- **Hoare triples** `{(P_i, C_i, Q_i)}` derived from the clause graph: `P_i` = conjunction of source‑node predicates, `C_i` = the connective (e.g., `if … then …`), `Q_i` = target‑node predicates.  

*Operations*  
1. **Structural parsing** (regex + shallow dependency): extract  
   - numeric comparisons (`>`, `<`, `=`),  
   - negations (`not`, `no`),  
   - conditionals (`if … then …`, `unless`),  
   - causal markers (`because`, `leads to`, `results in`),  
   - ordering (`before`, `after`).  
   Each yields a clause graph edge with base weight = 1.  
2. **Abductive hypothesis generation**: for each node with missing incoming edges, enumerate all single‑predicate completions consistent with domain vocab (≤ 5 per node). Score each hypothesis `h` by explanatory virtue:  
   `E(h) = Σ_{edges e satisfied by h} w_e – λ·|h|` (penalize hypothesis size). Keep top‑K hypotheses.  
3. **Hoare‑style constraint propagation**: treat each triple as a logical rule. Using forward chaining (modus ponens) propagate truth values from `P_i` through `C_i` to update `Q_i`. Implement with numpy matrix multiplication: `s' = clamp(s + A·s, 0,1)` where `A` encodes rule weights. Iterate to fixed point (≤ 10 steps).  
4. **Sensitivity analysis**: perturb the initial state vector `s₀` by ε‑ball (±0.1 on each known predicate) and recompute the final state `s_f`. Compute variance `V = Var(s_f)` across perturbations. Low variance → robust inference.  
5. **Scoring**: for each candidate answer `a`, map its predicates to a vector `v_a`. Score = `α·E(h*) – β·V + γ·sim(v_a, s_f)`, where `h*` is the best hypothesis, `sim` is cosine similarity (numpy), and α,β,γ are fixed weights (e.g., 0.4,0.3,0.3). Higher score → better answer.
