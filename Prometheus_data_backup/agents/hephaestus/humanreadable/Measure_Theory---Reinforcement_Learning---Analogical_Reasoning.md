## Nous Analysis

**Algorithm**  
We build a *belief‑measure* over a finite set of atomic propositions extracted from the text. Each proposition \(p_i\) corresponds to a grounded predicate (e.g., “X > Y”, “¬A”, “if C then D”). The belief is a probability vector \(\mathbf{b}\in\Delta^{N}\) (numpy array, sums to 1) representing a measure on the power set of propositions (the σ‑algebra is generated by the atoms).  

1. **Parsing → Graph**  
   - Regex patterns extract:  
     *Negations* (`not`, `no`),  
     *Comparatives* (`greater than`, `<`, `>`),  
     *Conditionals* (`if … then …`, `unless`),  
     *Causal* (`because`, `leads to`),  
     *Ordering* (`before`, `after`, `precedes`).  
   - Each extracted triple (subject, relation, object) becomes a directed edge labeled with the relation type; nodes are entity mentions or numeric literals. The resulting structure is a labeled directed graph \(G=(V,E,\ell)\).  

2. **Constraint Set**  
   From \(G\) we derive logical constraints:  
   - Transitivity for ordering edges (if \(a<b\) and \(b<c\) then \(a<c\)).  
   - Modus ponens for conditionals (if \(C\) and \(C\rightarrow D\) then \(D\)).  
   - Negation consistency (¬¬\(p\) ⇔ \(p\)).  
   - Numeric equality/inequality consistency.  
   Each constraint \(c_k\) is a clause over a subset of atoms; we store its satisfied‑indicator function \(s_k(\mathbf{x})\in\{0,1\}\) for a world \(\mathbf{x}\in\{0,1\}^N\).  

3. **Reinforcement‑Learning Update**  
   - State \(s\) = current belief vector \(\mathbf{b}\).  
   - Action \(a_i\) = flip the truth value of atom \(i\) (i.e., propose a new world \(\mathbf{x}'\) where \(x'_i=1-x_i\)).  
   - Immediate reward \(r(s,a_i)=\sum_k w_k\,[s_k(\mathbf{x}')-s_k(\mathbf{x})]\) where \(w_k\) are fixed weights (e.g., 1 for hard constraints, 0.5 for soft).  
   - Q‑table \(Q(s,a)\) is approximated linearly: \(Q(s,a)=\mathbf{b}^\top \theta_a\) with \(\theta_a\) a numpy vector; we update via stochastic gradient ascent:  
     \(\theta_a \leftarrow \theta_a + \alpha \big(r + \gamma \max_{a'} \mathbf{b'}^\top \theta_{a'} - \mathbf{b}^\top \theta_a\big)\mathbf{b}\).  
   - After \(T\) steps we obtain a refined belief \(\mathbf{b}^\*\).  

4. **Analogical Transfer**  
   A library of prototype graphs \(G^{(p)}\) with pre‑learned \(\theta^{(p)}\) is kept. For a new input we compute a graph‑matching score (Hungarian algorithm on node/edge label similarity) to retrieve the most analogous prototype \(p^\*\). Its \(\theta^{(p^\*)}\) initializes the learner, providing far‑transfer of constraint‑weighting patterns.  

5. **Scoring**  
   The final score for a candidate answer is the expected reward under the belief measure:  
   \(\displaystyle \text{Score}= \sum_{\mathbf{x}} \big(\prod_i b_i^{x_i}(1-b_i)^{1-x_i}\big) \sum_k w_k s_k(\mathbf{x})\).  
   This is computed efficiently by exploiting factorization (mean‑field approximation) using numpy.
