## Nous Analysis

**Algorithm: Evolving Epistemic Property‑Based Scorer (EEPBS)**  

*Data structures*  
- **Candidate graph** `G = (V, E)` where each node `v` is a proposition extracted from a candidate answer (e.g., “X causes Y”, “A > B”, “¬P”). Edges encode logical relations (implication, equivalence, contradiction) derived from syntactic patterns.  
- **Fitness vector** `f ∈ ℝ^k` where each dimension corresponds to an epistemic property: *justification* (presence of evidence clauses), *coherence* (internal consistency), *reliability* (frequency of supported patterns in a reference corpus), and *novelty* (distance from known facts).  
- **Mutation pool** `M` of small syntactic transformations (negation flip, quantifier swap, predicate substitution) generated by a property‑based tester (Hypothesis‑style) that shrinks to minimal failing inputs.  

*Operations*  
1. **Parsing** – Regex‑based extractor yields atomic clauses and tags for: negations, comparatives (`>`, `<`, `=`), conditionals (`if … then …`), causal verbs (`causes`, `leads to`), ordering relations (`before`, `after`). Each clause becomes a node; edges are added when a clause matches a rule schema (e.g., `A causes B` → edge `A → B` labeled *causal*).  
2. **Constraint propagation** – Using a Floyd‑Warshall‑style transitive closure on `G` we infer implied relations; contradictions are flagged when both `p → q` and `p → ¬q` appear.  
3. **Property‑based testing** – For each node we generate a set of mutant clauses via `M`. A mutant is *accepted* if it does not increase contradiction count and improves a surrogate epistemic score (e.g., adds a justification clause). The process iterates for a fixed budget (e.g., 50 generations), keeping the mutant with highest fitness – analogous to selection in evolution.  
4. **Scoring** – Final fitness `S = w·f_just + u·f_coh + v·f_rel + z·f_nov` where weights sum to 1. `f_just` = proportion of nodes with evidence tags; `f_coh` = 1 – (contradiction edges / total edges); `f_rel` = cosine similarity (numpy) between the candidate’s clause embedding (binary bag‑of‑relations) and a pre‑computed reliable‑pattern vector; `f_nov` = inverse of nearest‑neighbor distance to a set of known correct answers (lower distance → lower novelty).  

*Structural features parsed*  
Negations, comparatives, conditionals, causal claims, temporal ordering, existential/universal quantifiers, and conjunctive/disjunctive combinations.  

*Novelty*  
The triple blend is not found in existing scoring tools: evolutionary mutation‑selection loops are rarely coupled with property‑based shrinking and explicit epistemic property vectors. Most related work uses either pure logical theorem proving or statistical similarity; EEPBS adds a guided, fitness‑driven search over syntactic mutants, which is novel in the reasoning‑evaluation niche.
