**Algorithm: Typed Incentive‑Compatible Model Checker (TIMC)**  

1. **Parsing & Typing (Type Theory)**  
   - Input text is tokenized; regex patterns extract atomic propositions and attach simple types:  
     *`Prop`* for factual statements, *`Num`* for numeric literals, *`Ord`* for ordering relations, *`Cause`* for causal clauses.  
   - Each extracted clause becomes a term `t : τ` where τ ∈ {Prop, Num, Ord, Cause}. Dependent‑type‑like annotations record constraints (e.g., `Num(x) : x > 0`).  
   - The parsed structure is a directed acyclic graph **G** where nodes are typed terms and edges represent logical connectives (¬, ∧, →) extracted via pattern matching on cue words (“if”, “because”, “greater than”, etc.).

2. **Mechanism‑Design Layer (Incentive Compatibility)**  
   - Treat each candidate answer **a** as a strategy in a normal‑form game where the “principal” (the evaluator) desires the answer to satisfy a specification **Φ** (a set of temporal constraints derived from the question).  
   - Define utility `U(a, w) = 1` if the answer **a** makes **Φ** true in world **w**, else `0`. Worlds are generated by varying the truth values of atomic propositions consistent with the type constraints (e.g., a `Num` node must respect its numeric bounds).  
   - Compute the expected utility over all worlds using simple enumeration (state‑space size limited by the number of binary propositions; numpy arrays hold the truth‑value matrix).  
   - An answer is **incentive compatible** if no unilateral deviation to another candidate answer yields higher expected utility; this is checked by comparing utility vectors with numpy’s `max` operation.

3. **Model‑Checking Phase (Exhaustive Verification)**  
   - Convert the specification **Φ** into a linear‑time temporal logic (LTL) formula over the atomic propositions (e.g., `G (cause → effect)`, `F (value > threshold)`).  
   - Perform explicit‑state model checking on the Kripke structure built from the world enumeration: each state is a truth assignment; transitions are identity (static worlds) because we evaluate static logical consequences.  
   - Using numpy’s bitwise operations, propagate truth values through the LTL operators (next, until, globally) in a fix‑point loop; the result is a boolean array indicating which worlds satisfy **Φ**.  
   - The final score for answer **a** is the proportion of satisfying worlds weighted by the incentive‑compatibility indicator (1 if IC, 0 otherwise).
