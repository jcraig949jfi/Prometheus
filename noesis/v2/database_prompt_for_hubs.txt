PROMPT: Impossibility Theorem Hub Expansion for Noesis Database
Context
You are contributing to a mathematical structural analysis database (Project Prometheus / Noesis). The database decomposes mathematical systems into 11 primitives:
MAP, COMPOSE, REDUCE, EXTEND, COMPLETE, LIMIT,
SYMMETRIZE, BREAK_SYMMETRY, DUALIZE, LINEARIZE, STOCHASTICIZE
We have identified a meta-pattern called FORCED_SYMMETRY_BREAK:
COMPOSE(elements) → COMPLETE(closure) FAILS → forced BREAK_SYMMETRY(damage allocation)
This pattern occurs when a mathematical system attempts closure but an impossibility result prevents it, forcing every instantiation of that system to choose WHERE to allocate the structural damage. The taxonomy of damage-allocation strategies across traditions and domains is itself a structural object.
Known example: The Pythagorean comma. Twelve perfect fifths ≠ seven octaves (because no power of 2 equals any power of 3). Every tuning system in history must decide where to place the resulting error. Equal temperament distributes it uniformly. Meantone concentrates it. Pythagorean tuning isolates it in the wolf interval. Each is a BREAK_SYMMETRY resolution of the same impossibility.
Task
Identify impossibility results across all of mathematics, physics, engineering, social science, linguistics, and any other formal domain where:

A system attempts to satisfy a set of desirable properties simultaneously
A proven theorem or structural constraint makes simultaneous satisfaction impossible
Multiple distinct resolution strategies exist, each sacrificing different properties
The resolutions span multiple traditions, cultures, or historical periods where possible

Critical Instructions

RICH DESCRIPTIONS. Every resolution must have at minimum 3 sentences describing the structural mechanism. Thin descriptions are the #1 failure mode. Describe HOW the resolution handles the impossibility, not just WHAT it does.
CROSS-DOMAIN ANALOGS. For every resolution, identify at least one resolution in a DIFFERENT domain that uses the same damage-allocation strategy. Equal temperament distributes damage uniformly — what else distributes damage uniformly? This is where Noesis bridges come from.
PRIMITIVE SEQUENCES, NOT JUST SETS. Order matters. BREAK_SYMMETRY → MAP is different from MAP → BREAK_SYMMETRY. Specify the order in which primitives are applied in each resolution.
NOESIS SEARCH TARGETS. For each hub, identify where you think UNDISCOVERED resolutions might exist. What primitive combinations haven't been tried? What domains haven't confronted this impossibility but should have?
DO NOT INVENT. Every impossibility result must have a real formal proof or theorem behind it. Cite it. If a "tradeoff" is merely empirical or heuristic (no proven impossibility), mark it explicitly as CONJECTURED and explain what would need to be proven.
MARK EXISTING. If a system is already in the database (Pythagorean comma tuning systems, Arrow's theorem if present), mark it with "already_in_db": true so we can merge rather than duplicate.
ENUMERATE RESOLUTIONS EXHAUSTIVELY. Don't give 3 examples when 8 exist. The value is in the COMPLETE taxonomy of resolution strategies. Missing a resolution is missing a spoke.
ETHNOMATHEMATICAL AWARENESS. Many impossibilities were confronted independently by non-Western traditions. Mayan calendar systems resolve the same astronomical incommensurability as the Gregorian. Japanese well-temperament resolves the same comma as European temperaments. Aboriginal seasonal calendars resolve the same lunar-solar tension. Include these.

The goal is chain densification: turning a sparse graph (60 edges / 1,714 operations) into a richly connected knowledge structure where every impossibility result is a hub factory generating typed edges between previously unconnected domains.
