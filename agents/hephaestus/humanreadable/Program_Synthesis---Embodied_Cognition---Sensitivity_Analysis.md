## Nous Analysis

The algorithm builds a lightweight symbolic program from the text, grounds its primitives in sensorimotor‑style features, and scores answers by measuring how stable the program’s output is under small perturbations.

**Data structures**  
- `FeatureDict`: mapping from extracted primitives (e.g., `NUMBER`, `COMPARATIVE`, `NEGATION`, `CONDITIONAL`, `CAUSAL_VERB`, `ORDER_REL`) to typed values (float for numbers, bool for polarity, enum for direction).  
- `Clause`: a Horn‑clause tuple `(head, body)` where `head` and each body literal are `Predicate(name, args)`. Args refer to slots in `FeatureDict`.  
- `Program`: list of `Clause`s, limited to depth ≤ D (e.g., D = 3) and a maximum of C clauses (e.g., C = 4).  
- `PerturbationSet`: list of `FeatureDict`s generated by applying atomic perturbations (toggle negation, add/subtract ε to a number, flip ordering direction, swap antecedent/consequent of a conditional).

**Operations**  
1. **Structural parsing** – regexes extract: numeric literals (`\d+(\.\d+)?`), comparatives (`>`, `<`, `>=`, `<=`, `more than`, `less than`), negation cues (`not`, `no`, `never`), conditional patterns (`if … then …`, `when …`), causal verbs (`cause`, `lead to`, `result in`), temporal/spatial ordering (`before`, `after`, `above`, `below`), and quantifiers (`all`, `some`, `none`). Each match populates `FeatureDict`.  
2. **Program synthesis** – enumerate all possible Horn‑clause programs up to size C using a type‑directed search: predicates are limited to unary/binary relations whose argument types match the slots (e.g., `greater_than(NUMBER, NUMBER)`). A simple SAT‑style checker (implemented with `itertools.product` and Python’s `any`/`all`) verifies whether a program correctly maps the feature vector of a known correct answer to `True` and incorrect answers to `False` (if a small validation set is available); otherwise, the program is scored by its coverage of extracted logical patterns (more covered patterns → higher synthesis score).  
3. **Sensitivity‑based scoring** – for a candidate answer, compute its feature vector `f`. Run the synthesized program on `f` to get base confidence `c₀` (1 if head satisfied, else 0). Generate `k` perturbed vectors `fᵢ` from `PerturbationSet`; evaluate the program on each to get confidences `cᵢ`. Final score = `c₀ – λ * std([c₀, c₁, …, cᵢ])`, where λ ≈ 0.2 penalizes instability. Higher scores indicate answers that are both logically entailed and robust to small input changes.

**Structural features parsed**  
- Numerics and comparatives  
- Negation scopes  
- Conditional antecedents/consequents  
- Causal claim verbs  
- Temporal/spatial ordering relations  
- Quantifiers and conjunction/disjunction markers  

**Novelty**  
While program synthesis for rule induction, embodied grounding via sensorimotor features, and sensitivity analysis for robustness have each appeared separately, their tight coupling—using synthesized logical programs as the model whose output stability is measured after explicit, perturbation‑driven feature changes—has not been combined in existing QA scoring tools. Prior work either relies on neural similarity scores or static rule‑based entailment without perturbation‑based robustness checks.
