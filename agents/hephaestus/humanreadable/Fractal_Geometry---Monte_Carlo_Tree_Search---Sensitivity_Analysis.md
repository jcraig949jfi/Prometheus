## Nous Analysis

**Algorithm: Fractal‑Guided Monte Carlo Sensitivity Scorer (FG‑MCSS)**  

**Data structures**  
- *Parse tree*: each sentence is converted into a directed acyclic graph (DAG) where nodes are atomic propositions (e.g., “X > Y”, “¬P”, “if A then B”) and edges represent logical relations (implication, conjunction, negation). The DAG is stored as adjacency lists in NumPy arrays of dtype int32 for fast indexing.  
- *Search tree*: a mutable MCTS node holds (state = current sub‑DAG, visit count N, value sum V, prior P). Children are generated by applying one of three edit operations to the state: (1) flip a negation, (2) swap the arguments of a comparative, (3) perturb a numeric constant by ±ε drawn from a uniform distribution.  
- *Fractal cache*: a dictionary mapping a hash of a sub‑DAG to its Hausdorff‑like dimension estimate d = log (Nₑ)/log (1/δ), where Nₑ is the number of distinct edge types in the sub‑graph and δ is the recursion depth used when the sub‑DAG was first encountered. This provides a scale‑invariant complexity score that is recomputed only when the sub‑DAG changes.

**Operations**  
1. **Parsing** – regex extracts propositions, comparatives, conditionals, and numeric literals; builds the initial DAG.  
2. **Selection** – UCB1 selects child: Q + c·√(ln N_parent / N_child) where Q = V/N.  
3. **Expansion** – if the node is leaf and not terminal, generate all feasible edits (bounded to k = 3 per node) and add unexplored children.  
4. **Simulation (rollout)** – recursively apply random edits until a depth limit L = 5 is reached; at each step compute a *sensitivity* score: S = |Δoutput|/|Δinput| where Δoutput is the change in a simple truth‑table evaluation of the DAG (True/False) and Δinput is the magnitude of the edit (1 for logical flip, ε for numeric). The rollout returns the average S over the path.  
5. **Backpropagation** – update N, V ← V + S for each node on the path.  
6. **Fractal weighting** – after each backprop, multiply the node’s V by its cached fractal dimension d to reward structurally rich (self‑similar) sub‑graphs.  
7. **Scoring** – after a fixed budget of simulations (e.g., 2000), the root’s average value V/N is the final answer score; higher scores indicate answers whose internal logical structure is both stable under perturbations (low sensitivity) and exhibits fractal self‑similarity.
