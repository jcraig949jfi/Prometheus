## Nous Analysis

**Algorithm: Fractal‑Guided Counterfactual MCTS Scorer**

1. **Data structures**  
   - *Parse tree*: each sentence is converted into a directed acyclic graph (DAG) where nodes are atomic propositions (extracted via regex for negations, comparatives, conditionals, numeric thresholds, causal verbs, and ordering relations). Edges represent logical connectives (AND, OR, IMPLIES) and are weighted by a confidence ∈ [0,1] derived from cue‑word presence.  
   - *Fractal lattice*: the DAG is recursively subdivided into self‑similar sub‑graphs using a fixed depth D (e.g., D = 3). At each level ℓ, a node set Sₗ is generated by applying an iterated function system (IFS) that maps each node to its immediate children plus a stochastic perturbation ε∼Uniform(−0.05,0.05) to emulate Hausdorff‑scale variation.  
   - *MCTS nodes*: each MCTS node stores a sub‑graph Gₗ, a visit count N, and a value estimate Q (average reward). The root corresponds to the full parsed DAG of the candidate answer.

2. **Operations**  
   - *Selection*: UCB1 = Q + c·√(ln N_parent / N) with c = √2.  
   - *Expansion*: if the selected node’s sub‑graph has undeveloped child levels (ℓ < D), create a child node by applying one IFS mapping to produce a finer‑grained sub‑graph.  
   - *Simulation (rollout)*: perform a deterministic counterfactual walk: starting from the leaf sub‑graph, randomly flip the truth value of a leaf proposition (simulating a “do” intervention) and propagate changes through the DAG using modus ponens and transitivity (implemented via numpy matrix multiplication of the adjacency matrix). The rollout ends when a terminal condition is met (e.g., contradiction detected or max depth H = 5). Reward = 1 if the resulting world satisfies the prompt’s constraints (checked via regex‑extracted numeric/causal clauses), else 0.  
   - *Backpropagation*: update N and Q along the path.

3. **Scoring logic**  
   After a fixed budget B (e.g., 500 simulations), the scorer returns the root’s Q value as the candidate’s plausibility score. Higher Q indicates that, under many counterfactual perturbations, the answer remains consistent with the prompt’s logical and numeric constraints.

**Structural features parsed**  
- Negations (“not”, “no”) → polarity flags.  
- Comparatives (“greater than”, “less than”) → numeric inequality nodes.  
- Conditionals (“if … then …”) → implication edges.  
- Causal claims (“causes”, “leads to”) → do‑calculus style edges.  
- Ordering relations (“before”, “after”) → temporal edges.  
- Numeric values and units → leaf nodes with attached scalars for threshold checks.

**Novelty**

Prior work uses either MCTS for planning (e.g., game trees) or fractal metrics for similarity, but not both together with explicit do‑calculus rollouts for answer scoring.
