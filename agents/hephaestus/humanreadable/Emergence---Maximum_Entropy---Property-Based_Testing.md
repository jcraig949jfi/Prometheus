## Nous Analysis

**Algorithm: Entropy‑Guided Constraint‑Propagation Property Tester (ECP‑PT)**  

*Data structures*  
- **Atom graph** `G = (V, E)` where each node `v∈V` is a grounded proposition extracted from the prompt (e.g., “X > 5”, “¬rain”, “cause(Y,Z)”). Edges encode logical relations (implication, equivalence, ordering) derived via regex‑based pattern matching.  
- **Constraint matrix** `C ∈ ℝ^{|V|×k}` where each column corresponds to a measurable feature (truth value, numeric magnitude, temporal order). Initially `C` holds observed values from the prompt (1 for true literals, 0 for false, numeric constants as‑is).  
- **Weight vector** `w ∈ ℝ^{k}` representing the maximum‑entropy distribution over feature dimensions, initialized uniformly and updated by solving a convex optimization that maximizes entropy subject to linear constraints `C·w = μ`, where `μ` are empirical moments (e.g., average truth‑value of asserted propositions).  
- **Test suite** `T` generated by a property‑based tester: each property is a Horn clause derived from `G` (e.g., ∀x (A(x) → B(x))). The tester randomly instantiates variables with values sampled from the distribution defined by `w` (using numpy’s random choice for discrete, numpy.random.normal for continuous).  

*Operations*  
1. **Parsing** – regex extracts propositions, negations, comparatives, conditionals, causal predicates, and numeric constants; builds `G` and populates `C`.  
2. **Maximum‑entropy weighting** – iteratively adjusts `w` using numpy’s `linalg.lstsq` to satisfy moment constraints while maximizing `-∑ w_i log w_i`.  
3. **Constraint propagation** – applies forward chaining (modus ponens) and transitive closure on `G` using boolean matrix multiplication (`np.dot`) to derive implied nodes; updates `C` accordingly.  
4. **Property‑based testing** – for each Horn clause in `T`, samples `N` assignments from `w`, evaluates the clause using the current truth values in `C`, and records failures. A shrinking step reduces failing assignments by iteratively flipping literals with smallest impact on entropy (computed via `np.gradient`).  
5. **Scoring** – candidate answer `A` is parsed into its own atom graph `G_A`. The score is the proportion of properties in `T` that hold in `G_A` weighted by the entropy‑derived importance of each property: `score = Σ_i w_i * 1[property_i satisfied in G_A] / Σ_i w_i`.

No prior algorithm uses MaxEnt to derive a distribution over logical features that then drives a PBT‑style constraint‑propagation scorer, making ECP‑PT novel.
