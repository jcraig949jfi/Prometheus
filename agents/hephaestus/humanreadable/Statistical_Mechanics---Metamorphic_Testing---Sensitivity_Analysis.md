## Nous Analysis

**Algorithm: Ensemble‑Metamorphic Sensitivity Scorer (EMSS)**  

**Data structures**  
- `Prompt`: parsed into a list of *atomic propositions* `P = [p₁,…,pₙ]` where each `pᵢ` is a tuple `(type, value, scope)`. `type` ∈ {negation, comparative, conditional, numeric, causal, ordering}. `value` holds the extracted token or number; `scope` is the sentence index.  
- `Answer`: similarly parsed into `A = [q₁,…,qₘ]`.  
- `State vector` `s ∈ ℝᵏ` where each dimension corresponds to a proposition type count (e.g., `s[0]` = #negations, `s[1]` = #comparatives, …).  
- `Ensemble` `E = {s⁽ʲ⁾}` generated by applying *metamorphic perturbations* to the prompt (see below).  

**Operations**  
1. **Structural parsing** – deterministic regexes extract each proposition type and fill `P` and `A`. No ML; only `re` and string ops.  
2. **Metamorphic relation generation** – define a finite set `M` of input‑level mutations that preserve semantics under ideal reasoning:  
   - *Double numeric*: multiply every extracted number by 2.  
   - *Order‑swap*: reverse the order of two independent clauses (detected via coordinating conjunctions).  
   - *Negation flip*: insert/remove a leading “not” before a predicate.  
   For each `m ∈ M`, apply to the prompt → `P'ᵐ`, recompute its state vector `s'ᵐ`. Collect `E = {s'ᵐ | m∈M} ∪ {s₀}` where `s₀` is the original prompt vector.  
3. **Sensitivity propagation** – for each answer vector `a` (built from `A` similarly), compute the *ensemble deviation*:  
   \[
   D(a) = \frac{1}{|E|}\sum_{s' \in E}\|a - s'\|_2^2
   \]  
   This is the expected squared distance under a uniform ensemble, analogous to a partition‑function average in statistical mechanics.  
4. **Scoring** – lower deviation indicates the answer respects the metamorphic invariants; map to a score:  
   \[
   \text{score}(a) = \exp\bigl(-\beta \, D(a)\bigr)
   \]  
   with fixed inverse temperature `β = 1.0` (no learning). Scores are normalized to `[0,1]` across candidates.
