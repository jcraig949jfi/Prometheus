## Nous Analysis

**Algorithm – Symbiotic Hoare‑Property Checker (SHPC)**  

*Data structures*  
- **Clause graph** G = (V, E): each vertex V_i holds a parsed proposition (literal, comparative, conditional, numeric constraint). Edges E encode logical relations extracted by regex‑based parsers (e.g., “if A then B”, “A > B”, “¬C”).  
- **Invariant store** I: a set of Hoare‑style triples {P} C {Q} derived from the prompt, where P and Q are conjunctions of clauses in G and C is a program‑like step (often implicit: “assume X”, “derive Y”).  
- **Test‑case pool** T: generated by a property‑based tester that treats each clause in G as a property to be satisfied. For numeric clauses we sample from domains; for relational clauses we generate variable assignments that respect ordering or equality constraints.  
- **Shrinking queue** Q: holds failing assignments ordered by a size metric (e.g., sum of absolute numeric deviations, number of violated literals).  

*Operations*  
1. **Parse** the prompt and each candidate answer into clause graphs G_prompt and G_ans using deterministic regexes for negations, comparatives (“>”, “<”, “=”), conditionals (“if … then …”), causal markers (“because”, “leads to”), and numeric literals.  
2. **Build invariants**: for each conditional in G_prompt, create a Hoare triple {antecedent} [assume] {consequent}. Add these to I.  
3. **Property‑based testing**: treat each clause in G_ans as a property. Generate random assignments that satisfy I (using rejection sampling guided by the invariant store). Evaluate the property; if falsified, push the assignment onto Q.  
4. **Shrink**: repeatedly pop from Q, apply delta‑debugging (remove literals, reduce numeric magnitude) to find a minimal failing input. If no failing input remains after shrinking, the answer passes; otherwise record the minimal counter‑example.  
5. **Score**:  
   - **Base score** = 1 – (|Q_min| / |T|), where |Q_min| is the number of shrunk failing cases and |T| the total generated tests.  
   - **Penalty** for each unresolved invariant in I (i.e., a conditional whose consequent cannot be derived from the antecedent under the answer’s clauses).  
   - Final score = base – λ·penalty, λ = 0.2 (tuned to keep score in [0,1]).
