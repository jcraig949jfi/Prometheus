## Nous Analysis

**Algorithm**  
The tool builds a hybrid constraint‑reasoning engine that treats each candidate answer as a set of temporal‑logic propositions extracted from the prompt. First, a lightweight parser (regex‑based for efficiency) converts sentences into atomic predicates \(p_i\) and links them with logical operators (¬, ∧, ∨, →) and arithmetic constraints (>, <, =, ≤, ≥). The resulting formula is stored in conjunctive normal form (CNF) as a list of clause objects, each clause holding a set of literal IDs and, for arithmetic literals, a coefficient vector and bound.

A belief state over the Boolean variables is maintained as a Gaussian mean \(\mu\in[0,1]^n\) (probability of truth) and covariance \(\Sigma\) (uncertainty). This is the Kalman‑filter‑style state: prediction step propagates \(\mu\) through deterministic logical constraints using unit‑propagation (if a clause becomes unit, the literal’s mean is set to 1 or 0 and variance to 0). The update step incorporates evidence generated by property‑based testing: the Hypothesis‑style generator creates random assignments to the free variables, evaluates the CNF, and when a clause fails, the shrinking algorithm reduces the failing assignment to a minimal counterexample. The counterexample provides a binary observation vector \(z\) (1 = literal satisfied, 0 = violated) with observation noise modeled as isotropic Gaussian \(R=\sigma^2 I\). The Kalman update equations  
\[
K = \Sigma H^T (H\Sigma H^T + R)^{-1},\quad
\mu \leftarrow \mu + K(z - H\mu),\quad
\Sigma \leftarrow (I - KH)\Sigma
\]  
are applied, where \(H\) selects the literals appearing in the failed clause. After processing a batch of tests, the score for a candidate answer is the negative log‑likelihood \(-\log \mathcal{N}(z; H\mu, H\Sigma H^T + R)\), lower scores indicating higher consistency with the parsed constraints.

**Structural features parsed**  
- Negations (¬) → literal polarity.  
- Comparatives and ordering relations (>, <, ≥, ≤, =) → arithmetic literals.  
- Conditionals (if‑then) → implication clauses transformed to CNF.  
- Causal claims → treated as implication with optional temporal index.  
- Numeric values → coefficients in arithmetic literals.  
- Quantifier‑free existential/universal patterns → captured via generated test assignments.

**Novelty**  
Each component exists separately (Kalman filters for state estimation, property‑based testing for falsification, SAT/SMT for satisfiability). Their tight coupling—using Kalman belief updates driven by shrinking counterexamples from property‑based tests on a SAT‑encoded formula—has not been reported in the literature, making the combination novel.
