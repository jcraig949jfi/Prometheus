## Nous Analysis

**Algorithm**  
We build a hybrid scorer that treats each candidate answer as a hypothesis *h* in an active‑inference loop, whose predicted observations are generated by a fixed‑random recurrent reservoir, and whose readout weights are continuously adapted by a recursive‑least‑squares (adaptive‑control) rule to minimise expected free energy.

1. **Reservoir encoding** – For each token *t* in the parsed structural feature sequence (see §2) we draw a one‑hot vector *xₜ*∈ℝᴰ (D≈500). The reservoir state *sₜ*∈ℝᴺ (N=200) evolves as  
   \[
   s_{t}= \tanh\!\big(W_{\text{in}}x_{t}+W_{\text{rec}}s_{t-1}\big),
   \]  
   where *W_in*∈ℝᴺˣᴰ and *W_rec*∈ℝᴺˣᴺ are fixed random matrices (spectral radius < 1). The final state *s_T* is the reservoir’s representation of the input.

2. **Readout (adaptive control)** – A weight matrix *W_out*∈ℝᴷˣᴺ maps reservoir states to a K‑dimensional observation vector *ŷ* = *W_out s_T* (K equals the number of observable features, e.g., polarity, magnitude, order). *W_out* is updated online for each candidate answer using the recursive‑least‑squares update:  
   \[
   K_t = P_{t-1}s_T\bigl(\lambda + s_T^\top P_{t-1}s_T\bigr)^{-1},\quad
   W_out \leftarrow W_out + K_t\bigl(y - \hat y\bigr)^\top,
   \]  
   \[
   P_t = \lambda^{-1}\bigl(P_{t-1} - K_T s_T^\top P_{t-1}\bigr),
   \]  
   with forgetting factor λ≈0.99 and *y* the ground‑truth feature vector derived from the question.

3. **Active‑inference scoring** – Expected free energy for hypothesis *h* (candidate answer) is approximated as  
   \[
   G(h) = \underbrace{\tfrac12\|y-\hat y_h\|^2}_{\text{accuracy}} + \underbrace{\tfrac12\,\text{Tr}\!\big(\Sigma_h\big)}_{\text{complexity}},
   \]  
   where \(\hat y_h = W_out s_T^{(h)}\) is the prediction conditioned on the answer’s feature vector and \(\Sigma_h\) is the posterior covariance approximated by the inverse of the Fisher information (diagonal, computed from the reservoir Jacobian). Lower *G* → higher score; we rank candidates by \(-G(h)\).
