**Algorithm**  
1. **Parsing & proposition extraction** – Using a handful of regex patterns we pull atomic propositions from the prompt and each candidate answer:  
   - *Negations*: `\bnot\b|\bno\b|\bnever\b`  
   - *Comparatives*: `\bmore\b|\bless\b|\bgreater\b|\blesser\b`  
   - *Conditionals*: `if\s+.+?then\s+`  
   - *Causal claims*: `\bbecause\b|\btherefore\b|\bdue to\b`  
   - *Ordering relations*: `\bbefore\b|\bafter\b|\b>\b|\b<\b`  
   - *Numeric values*: `\d+(\.\d+)?`  
   Each proposition is stored as a tuple `(type, polarity, args)` where `type` ∈ {neg, comp, cond, caus, order, num} and `polarity` = +1 for affirmed, –1 for negated.  

2. **Feature vector construction** – For each text we build a sparse binary vector **x** ∈ {0,1}^D where D is the number of distinct proposition templates observed across all candidates (e.g., “neg‑comp”, “caus‑order”). The entry is 1 if the template appears with its polarity.  

3. **Dialectical thesis‑antithesis‑synthesis** –  
   - *Thesis* = feature vector of the candidate (**x_c**).  
   - *Antithesis* = vector generated by flipping the polarity of every proposition that appears in a contradiction set C (detected when both a proposition and its negation are present in the same text).  
   - *Synthesis* = result of applying constraint‑propagation (transitivity of order, modus ponens on conditionals) to the union of thesis and antithesis, yielding a corrected vector **x_s**.  

4. **Matched‑filter scoring** – Estimate noise covariance Σ from the variance of each feature across all candidates (diagonal Σ for simplicity). The matched filter output for candidate c is:  
   \[
   s_c = \frac{{\bf x}_s^{\top}\Sigma^{-1}{\bf x}_r}{\sqrt{{\bf x}_s^{\top}\Sigma^{-1}{\bf x}_s}}
   \]  
   where **x_r** is the reference solution vector (built from a human‑provided answer key). This maximizes the signal‑to‑noise ratio between the synthesized answer and the key.  

5. **Multi‑armed bandit allocation** – Treat each candidate as an arm. After an initial round of computing s_c, we maintain an upper‑confidence bound:  
   \[
   UCB_c = s_c + \alpha \sqrt{\frac{\ln N}{n_c}}
   \]  
   where n_c is the number of times we have deep‑parsed candidate c (initially 1), N is total rounds, and α controls exploration. In each iteration we select the arm with highest UCB, increase its parsing depth (add higher‑order n‑gram proposition templates), recompute **x_c**, **x_s**, and s_c, then update n_c. After a fixed budget B (e.g., B = 5× number of candidates) we stop and return the final s_c as the score.
