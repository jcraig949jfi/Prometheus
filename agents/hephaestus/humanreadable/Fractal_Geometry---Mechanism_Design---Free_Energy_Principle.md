## Nous Analysis

The algorithm builds a **fractal constraint graph** from the prompt, treats each candidate answer as a proposed mechanism, and scores it by the **variational free energy** of the mismatch between the answer’s constraints and the prompt’s constraints.

1. **Data structures**  
   - Token list `txt` from the prompt.  
   - Relation dictionary `R` where each key is a tuple `(i, j, type)` indicating a directed edge from token index `i` to `j` with type ∈ {neg, comp, cond, caus, ord, num}.  
   - NumPy array `feat` of shape `(n_rel, 6)` holding binary flags for each relation type and, for comparatives and numerics, the extracted numeric value.  
   - For each candidate answer `a`, we build an analogous feature array `feat_a` by the same regex extraction.

2. **Operations**  
   - **Regex extraction** (using only `re`) finds patterns: `\bnot\b` (neg), `\b(>|<|>=|<=|==)\b` (comp), `\bif\b.*\bthen\b` (cond), `\bbecause\b|\bleads to\b` (caus), `\bbefore\b|\bafter\b` (ord), `\d+(\.\d+)?` (num).  
   - Construct adjacency matrices `A_type` for each relation type; compute transitive closure via repeated Boolean matrix multiplication (numpy `@`) to propagate constraints (modus ponens, transitivity).  
   - **Constraint violation vector** `v = feat_a - feat` (element‑wise; for binary flags mismatch = 1, for numerics = absolute difference).  
   - **Energy** `E = np.sum(v**2)` (squared L2 norm, penalizes mismatched logical structure).  
   - **Entropy** approximates uncertainty over alternative parses: count distinct parse trees generated by varying greedy vs. lazy regex matches; obtain probabilities `p_k` and `H = -np.sum(p_k * np.log(p_k + 1e-12))`.  
   - **Free energy** `F = E + H`.  
   - **Score** `S = -F` (lower free energy → higher score). The mechanism design view: the answer is “incentive compatible” when it minimizes the predictor’s surprise.

3. **Structural features parsed**  
   Negations, comparatives, conditionals, causal claims, ordering relations (before/after), numeric values, and quantifiers (via `\b all\b|\b some\b|\b none\b`).
