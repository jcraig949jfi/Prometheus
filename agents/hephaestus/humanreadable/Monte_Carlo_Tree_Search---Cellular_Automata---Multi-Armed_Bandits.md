**Algorithm**  
1. **Parsing stage** – Using only regex (re) we extract from the prompt and each candidate answer a list of atomic propositions *pᵢ* = (subject, relation, object, polarity). Each proposition receives a unique integer ID. We also extract binary constraints:  
   - *Implication*: “if A then B” → (A → B)  
   - *Negation*: “not A” → (¬A)  
   - *Comparative*: “X > Y” → (gt(X,Y))  
   - *Causal*: “A causes B” → (cause(A,B))  
   - *Ordering*: “before/after” → (before(A,B))  
   These constraints are stored in a static list *C*.

2. **State representation** – A node *n* in the Monte Carlo Tree Search (MCTS) holds a bit‑vector *Sₙ* of length *P* (number of propositions) indicating which propositions are currently assumed true (1) or false (0). The root starts with all bits = 0 (unknown).

3. **Cellular Automata update** – Given a state *S*, we apply a synchronous CA rule set derived from *C*:  
   For each constraint *c* in *C* we compute its truth value under *S* (using numpy logical ops). If *c* is violated we flip the minimal‑cost literal(s) to restore satisfaction (cost = 1 per flipped bit). This yields a deterministic transition *S' = CA(S)*. The process iterates until a fixed point or a max of *k* sweeps (k=5). The resulting fixed point *Ŝ* is the maximally consistent interpretation reachable from *S* under the local rules.

4. **Reward (rollout)** – After reaching a fixed point, the reward *r* is:  
   r = (# satisfied constraints) – λ·(# flipped bits)  
   where λ=0.2 penalizes arbitrary flips. This is a scalar numpy float.

5. **MCTS with Bandit selection** – For each node we keep:  
   - *Nₙ*: visit count  
   - *Qₙ*: average reward of rollouts originating from *n*  
   Child nodes are generated by **expansion**: picking one proposition not yet fixed in *Sₙ* and toggling its bit (creating two children: true/false).  
   Selection uses UCB1: choose child *c* maximizing Q_c + α·sqrt(log Nₙ / N_c) (α=1.0).  
   After selection we simulate a rollout: repeatedly apply CA until fixed point, then reward as above.  
   Back‑propagation updates N and Q along the path.

6. **Scoring** – After a fixed simulation budget *B* (e.g., 2000 iterations) the score for a candidate answer is the *Q* value of the root node (average reward of all explored trajectories). Higher scores indicate answers that are more consistent with the extracted logical structure under the CA‑derived inference.
