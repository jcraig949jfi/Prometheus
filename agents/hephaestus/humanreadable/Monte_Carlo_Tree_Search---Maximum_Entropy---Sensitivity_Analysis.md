## Nous Analysis

**Algorithm**  
1. **Parsing & proposition extraction** – Using a handful of regex patterns we pull from the prompt and each candidate answer:  
   * atomic propositions `P_i` (subject‑predicate‑object triples),  
   * polarity `p_i∈{+1,‑1}` for negation,  
   * comparative type `c_i∈{<,>,=,≠}` with numeric value `v_i` when present,  
   * conditional antecedent/consequent flags,  
   * causal direction flag.  
   Each proposition is stored as a struct `{id, features∈ℝ^k}` where the feature vector encodes the above binary/categorical cues (one‑hot for type, scalar for `v_i`). All propositions from prompt + answer form a matrix `X∈ℝ^{n×k}`.

2. **Maximum‑Entropy weighting** – We treat the weight vector `w∈ℝ^k` as the parameters of a log‑linear model `P(z|X,w) ∝ exp(w·(X·z))` where `z∈{0,1}^n` is a truth‑assignment vector. The empirical feature expectation from the prompt is `\hat{μ}= (1/m) Σ_{prompt propositions} X_row`. We find `w` that maximizes entropy subject to `E_{P}[X·z]=\hat{μ}` using iterative scaling (numpy only):  
   ```
   w ← 0
   repeat:
       μ ← Σ_z P(z|X,w) (X·z)   # approximated by sampling 10k assignments
       w ← w + η * log(\hat{μ}/μ)   # element‑wise
   ```
   The resulting `w` is the least‑biased distribution consistent with the prompt’s observed features.

3. **Monte‑Carlo Tree Search over worlds** –  
   *State*: current assignment `z`.  
   *Selection*: UCB1 on child nodes generated by flipping a single proposition’s truth value (respecting hard constraints such as transitivity of comparatives; these are enforced by a deterministic propagation step after each flip).  
   *Expansion*: add all legal single‑flip children not yet visited.  
   *Rollout*: from a leaf, randomly flip propositions for depth `D=5`, compute utility `U(z)= w·(X·z)`.  
   *Back‑propagation*: update visit count `N` and total utility `Q` for each node.  
   After `S` simulations (e.g., 2000), the score for the answer is `Q(root)/N(root)`, i.e., the expected utility under the MaxEnt‑weighted world distribution explored by MCTS.

4. **Sensitivity analysis** – The gradient of the expected utility w.r.t. `w` is simply the expected feature counts `E[X·z]` gathered during rollouts. We report this vector alongside the score to indicate which linguistic features (negation, comparative magnitude, causal direction, etc.) most influenced the rating.
