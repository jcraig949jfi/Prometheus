## Nous Analysis

The algorithm treats each candidate answer as a root node in a Monte Carlo Tree Search (MCTS) whose children are generated by a dialectical expansion rule.  
A node stores:  
- ** propositions **: a list of atomic propositions extracted via regex (e.g., “X > Y”, “¬P”, “if A then B”).  
- ** constraint matrix ** C (numpy bool [N×N]) where C[i,j]=1 indicates a derivable relation (transitivity, modus ponens, numeric equality/inequality) between proposition i and j.  
- ** visit count ** n and ** total value ** v.  

**Selection**: choose child with highest UCB = v/n + c·√(ln N_parent / n).  
**Expansion (dialectic step)**: from the selected node produce three children:  
1. **Thesis** – the node as‑is (literal meaning).  
2. **Antithesis** – for each proposition, create a negated copy (flip polarity, invert comparatives, swap antecedent/consequent in conditionals).  
3. **Synthesis** – apply resolution: pair a thesis proposition with an antithesis proposition that are complementary (e.g., P and ¬P) and replace them with their logical consequence (derived via C).  
Each child’s proposition list is re‑parsed, and C is updated by running a deterministic constraint‑propagation pass (transitive closure, modus ponens, numeric range checks).  

**Rollout**: randomly walk the constraint graph, applying forward chaining until no new propositions arise; count satisfied constraints (reward +1 each) and violated constraints (penalty −1 each).  

**Backpropagation**: update n and v of all nodes on the path with v ← v + rollout_score.  

After a fixed budget of simulations, the final score for an answer is the average v of its root node (lower intrinsic cognitive load → higher v because fewer propositions → denser C → more satisfied constraints). Cognitive load is approximated as L = α·|props| + β·|redundant_clauses| (extraneous) – γ·|synthesis_edges| (germane), and the rollout_score is subtracted by λ·L to penalize overload.
