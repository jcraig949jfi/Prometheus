## Nous Analysis

**Algorithm**  
We build a Monte Carlo Tree Search (MCTS) whose nodes encode a *structural feature graph* of a candidate answer. Each node stores:  

* `feat_vec` – a NumPy array of binary flags for parsed linguistic structures (negation, comparative, conditional, causal, ordering, quantifier, numeric value, coreference).  
* `children` – list of child nodes generated by analogical variation.  
* `value` – accumulated reward estimate.  
* `visits` – MCTS visit count.  

The tree is guided by a *gene regulatory network* (GRN) that treats each feature as a gene. The GRN adjacency matrix `W` (size F×F, initialized from known linguistic constraints, e.g., “comparative → ordering”, “causal → conditional”) holds activation (+1) or inhibition (‑1) weights. During a simulation (rollout) we start from the node’s `feat_vec` and iteratively update activation `a ← sigmoid(W @ a + b)` for a fixed number of steps (using only NumPy). The resulting activation pattern predicts how well the feature set satisfies constraints; the simulation reward is the dot‑product between the final activation and a *goal vector* derived from the prompt’s feature graph (higher when prompt and answer share the same relational structure).  

Selection uses the UCB1 formula: `value/visits + C * sqrt(log(parent.visits)/visits)`. Expansion creates children by applying *analogical operators*: copy the parent’s feature graph, then randomly map a subset of relations to analogous relations in another domain (e.g., swap “greater‑than” with “later‑than”) using a pre‑built analogy table (WordNet‑style hypernym/hyponym pairs). Backpropagation updates `value += reward` and `visits += 1` on the path to the root. After a fixed budget of iterations, the score for a candidate answer is the average reward of its root node (or the maximal value among its descendants).
