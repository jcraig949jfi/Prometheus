   `M_i: T_i(x) → y_i` where `T_i` is a deterministic transformation (e.g., double all numbers, swap order of two comparatives, negate a clause). For each MR we compute the transformed feature vector `x' = T_i(x)` and the corresponding candidate answer vector `y'` (obtained by applying the same transformation to the answer’s text).  

3. **Fitness evaluation** –  
   *Constraint satisfaction*: `c_i = 1` if `y'` obeys the expected MR (checked via simple rule: e.g., if numbers doubled then numeric entries in y' must be ≈2× original; if a negation flipped then the negation flag toggles).

   Overall fitness: `F = Σ w_i·c_i – λ·L`, with weights `w_i` (default 1) and λ a small regularizer (0.1).  

4. **Genetic optimisation** – Initialise a population of N=30 answer vectors (randomly sampled from the candidate set or generated by mutating the original answer). Iterate for G=20 generations:  
   *Selection*: tournament size 3 based on F.  
   *Crossover*: uniform swap of feature subsets.  
   *Mutation*: flip a random negation flag, add Gaussian noise (σ=0.05) to a numeric feature, or toggle a comparative direction.  
   The elite vector (highest F) is returned as the scored answer; its fitness value is the final score (higher = better).
