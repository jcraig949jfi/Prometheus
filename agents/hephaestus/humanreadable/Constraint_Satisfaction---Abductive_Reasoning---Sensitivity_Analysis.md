## Nous Analysis

**Algorithm: Constraint‑Driven Abductive Sensitivity Scorer (CDASS)**  

*Data structures*  
- **Literal graph** `G = (V, E)`: each node `v` is a propositional literal extracted from the prompt or a candidate answer (e.g., “X > 5”, “¬rain”, “cause(Y,Z)”). Edges encode binary relations (comparative, causal, temporal) derived from regex patterns.  
- **Constraint store** `C`: a set of clauses in conjunctive normal form (CNF) built from the prompt’s hard constraints (e.g., “if A then B”, “exactly one of {P,Q,R}”). Each clause is represented as a list of integer literals (positive for affirmed, negative for negated).  
- **Hypothesis pool** `H`: a list of candidate explanations generated by abductive step; each hypothesis `h` is a subset of literals not present in the prompt but consistent with `C`.  
- **Sensitivity matrix** `S`: `|H| × |V|` numpy array where `S[i,j]` measures the change in satisfaction score of hypothesis `i` when literal `j` is flipped (perturbed).  

*Operations*  
1. **Parsing** – regex extracts literals and binary constraints; builds `G` and initial `C`.  
2. **Unit propagation** – enforces arc consistency on `C` (AC‑3) using a queue; yields a reduced constraint set `C'`.  
3. **Abductive generation** – for each unsatisfied clause in `C'`, compute its minimal hitting set of missing literals (using a simple greedy set‑cover). Add each hitting set as a hypothesis `h` to `H`.  
4. **Scoring** – for each hypothesis `h`, compute a base satisfaction score `sat(h) = # satisfied clauses in C' ∪ h / # clauses`.  
5. **Sensitivity analysis** – perturb each literal in `V` (toggle its truth value) and recompute `sat`; store delta in `S`. The robustness score `rob(h) = 1 - (mean(|S[h,:]|) / max_possible_delta)`.  
6. **Final score** – `score(h) = α·sat(h) + (1-α)·rob(h)` with α=0.6 (tunable). Return the hypothesis with highest score; candidate answers are ranked by the score of the hypothesis they best support.  

*Structural features parsed*  
- Negations (`not`, `¬`) → negative literals.  
- Comparatives (`>`, `<`, `≥`, `≤`, `better than`) → ordered numeric constraints.  
- Conditionals (`if … then …`, `unless`) → implication clauses.  
- Causal verbs (`cause`, `lead to`, `results in`) → directed edges with a causal predicate.  
- Temporal markers (`before`, `after`, `while`) → precedence constraints.  
- Quantifiers (`all`, `some`, `none`) → cardinality constraints translated to CNF via standard encoding.  

*Novelty*  
The triple‑layer loop (constraint propagation → abductive hypothesis generation → sensitivity‑based robustness) is not found in existing pure‑numpy reasoning scorers. Prior work uses either SAT‑based validation or similarity‑based ranking; CDASS uniquely couples explanatory generation with perturbation analysis to penalize fragile hypotheses, a combination absent from current open‑source toolkits.
