## Nous Analysis

**1. Algorithm**  
We treat each candidate answer as a noisy observation of an underlying “reasoning signal” – the set of logical relations that a correct answer must exhibit. The pipeline has three coupled stages:

*Signal extraction (symbiosis)* – Parse the prompt and each candidate with a fixed set of regex patterns that capture atomic propositions and their logical connectors (negation, comparative, conditional, causal, numeric, ordering). From each text we build a binary feature vector **f** ∈ {0,1}^K where K is the number of distinct relation types (e.g., “¬P”, “P > Q”, “if P then Q”, “P causes Q”, numeric equality, transitive chain). The parsing stage is mutual: the set of patterns is refined iteratively by keeping only those that appear in ≥ τ % of the top‑scoring candidates (the “benefit” shared between parser and scorer).

*Matched filtering* – Maintain a reference vector **r** that represents the ideal reasoning pattern for the question (derived from a small set of human‑annotated exemplars or generated by rule‑based template). For each candidate we compute the normalized cross‑correlation  
\( s = \frac{ \langle f, r \rangle }{ \|f\|\|r\| } \)  
using numpy dot products. This yields a detection score that is maximal when the candidate’s logical structure aligns with the reference.

*Multi‑armed bandit allocation* – Each candidate is an arm. After an initial round of scoring with the matched filter, we compute an Upper Confidence Bound  
\( \text{UCB}_i = s_i + \alpha \sqrt{ \frac{\ln N}{n_i} } \)  
where *s_i* is the filter score, *n_i* the number of times arm *i* has been evaluated, *N* total evaluations, and α a exploration constant. The arm with the highest UCB is selected for a second, more expensive check: constraint propagation (transitivity, modus ponens) over the parsed logical graph to detect violations. Violations subtract a penalty *p* from *s_i*. The updated score becomes the arm’s reward, and the bandit statistics are refreshed. The process repeats for a fixed budget or until UCB convergence. The final score for each candidate is the last *s_i − p*.

**3. Novelty**  
Matched filtering is standard in signal processing; multi‑armed bandits are ubiquitous in online learning; structural parsing of logical relations appears in some SAT‑style solvers and natural‑language inference tools. The tight coupling — using bandit‑driven exploration to decide when to invoke costly constraint propagation on the parsed logical graph, while the parser itself is refined by the mutual‑benefit (symbiosis) criterion — has not been reported in the literature. Hence the combination is novel for reasoning‑answer scoring.
