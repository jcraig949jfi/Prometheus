**Algorithm**  
We maintain a population \(P\) of candidate answer representations. Each representation is a fixed‑length NumPy vector \(\mathbf{x}\in\mathbb{R}^D\) built from binary structural features (see §2). The algorithm proceeds in generations:

1. **Initialization** – Randomly sample \(N\) vectors from a uniform distribution \([0,1]^D\).  
2. **Evaluation (Multi‑Armed Bandit)** – For each individual \(i\) we compute a *scale‑dependent loss* \(L_i^{(s)}\) at renormalization scale \(s\) (see below). The bandit treats each individual as an arm; we allocate evaluation budget using Upper Confidence Bound:  
   \[
   \text{UCB}_i = \overline{L}_i + c\sqrt{\frac{\ln t}{n_i}},
   \]  
   where \(\overline{L}_i\) is the mean loss over observed scales, \(n_i\) the number of times arm \(i\) was sampled, and \(t\) the total evaluations so far. The arm with lowest UCB is selected for the next evaluation step.  
3. **Renormalization (Coarse‑graining)** – For a selected individual we compute loss at a hierarchy of scales \(s=0,1,2\). At scale \(s\) we pool features by averaging over non‑overlapping blocks of size \(2^s\) (using `numpy.reshape` and `mean`). The loss at scale \(s\) is the normalized Hamming distance between the pooled vector of the candidate and that of a reference answer (provided with the prompt). The total loss is a weighted sum  
   \[
   L_i = \sum_{s} w_s L_i^{(s)},
   \]  
   where the weights \(w_s\) are updated after each generation by simple reinforcement: \(w_s \leftarrow w_s + \eta (L_i^{\text{best}}-L_i^{(s)})\) (projected to the simplex).  
4. **Evolutionary Update** – The top \(k\) individuals are kept as elites. The rest of the population is generated by mutation (flip each bit with probability \(p_m\)) and crossover (uniform blend of two parents) using NumPy’s random functions.  
5. **Iterate** – Steps 2‑4 repeat for a fixed number of generations or until convergence of the elite loss. The final score for a candidate answer is the elite’s normalized loss (lower = better).
