**Algorithm – MCTS‑Matched‑Filter‑Metamorphic Scorer**  
The scorer builds a Monte‑Carlo Tree Search where each node represents a *feature vector* extracted from a candidate answer.  
1. **Feature extraction (prompt‑driven signal):** From the prompt we derive a reference signal `s` (numpy array) that encodes expected structural patterns: presence/absence of negations, comparatives, conditionals, numeric constants, causal verbs, and ordering tokens. Extraction uses regexes to produce a binary vector `f ∈ {0,1}^k` (k ≈ 20).  
2. **Node structure:**  
   ```python
   class Node:
       vec: np.ndarray          # feature vector of the node’s answer variant
       N: int                   # visit count
       W: float                 # accumulated value (matched‑filter score)
       children: List[Node]     # generated by metamorphic mutations
   ```  
   The root node holds the vector of the raw candidate answer.  
3. **Selection (UCB1):** For a node `n`, choose child `c` maximizing  
   `c.W/c.N + sqrt(2 * ln(n.N) / c.N)` if `c.N>0`, else infinite.  
4. **Expansion (Metamorphic mutation):** From a selected node, generate children by applying a fixed set of metamorphic relations to its feature vector:  
   *Negation flip* (`¬p`), *comparative swap* (`>`↔`<`), *conditional antecedent/consequent swap*, *numeric scaling* (×2, ÷2), *ordering reversal* (first↔last). Each mutation yields a new vec `v'` and a child node.  
5. **Simulation (Matched filtering):** For a leaf node, compute the matched‑filter score  
   `z = np.dot(s, v_leaf) / (np.linalg.norm(s)*np.linalg.norm(v_leaf))`  
   (cosine similarity, i.e., optimal detection of known signal in noise). This `z∈[‑1,1]` is the simulation reward.  
6. **Backpropagation:** Increment `N` and add `z` to `W` for all nodes on the path back to the root.  
7. **Score:** After a fixed budget of simulations (e.g., 2000), the estimator `root.W / root.N` is the final answer score.
