## Nous Analysis

**Algorithm**  
1. **Parsing (System 1‑style fast heuristic)** – Using regular expressions we extract from the prompt and each candidate answer a set of literals L and binary relations R:  
   *Negation* (`not`, `no`), *comparatives* (`>`, `<`, `≥`, `≤`, `more than`, `less than`), *conditionals* (`if … then …`, `unless`), *causal* (`because`, `due to`), *temporal ordering* (`before`, `after`, `while`).  
   Each literal gets an integer id; each relation becomes a directed edge labeled with its type (implication, equivalence, ordering, etc.). The graph is stored as two NumPy arrays: `src` (source node ids) and `dst` (target node ids) plus a dtype‑structured array `edge_type`.  

2. **Constraint encoding (System 2‑style slow deliberation)** – From the prompt we build a CNF clause set Φ₀: each extracted literal is a unit clause; each implication `A → B` becomes the clause `¬A ∨ B`; comparatives and orderings are encoded as difference constraints transformed to propositional atoms via threshold encoding (e.g., `x > 5` → atom `gt_x_5`).  

3. **Propagation & conflict detection** – We run a lightweight DPLL unit‑propagation loop on Φ₀ ∪ Φₐ (where Φₐ is the CNF of the answer). Propagation yields:  
   *SAT flag* = 1 if no contradiction, else 0.  
   *Unsat core size* ≈ number of literals forced to both true and false during propagation (a proxy for minimal unsatisfiable core).  

4. **Network‑science scoring** – On the answer’s relation graph we compute (using only NumPy):  
   *Clustering coefficient* C (average of local triangles / possible triangles).  
   *Average shortest‑path length* L (via repeated BFS on the unweighted graph; we approximate with Floyd‑Warshall on the adjacency matrix for ≤ 200 nodes, still O(n³) but fine for short answers).  
   *Small‑world score* = C / C_rand ÷ (L / L_rand) where random baselines are generated by shuffling edges.  

5. **Final score** –  
   `S = w1·SAT + w2·(1 – unsat_core/|L|) + w3·C_norm + w4·(1 – L_norm) + w5·Jaccard(System1)`  
   where the Jaccard term is the fast System 1 overlap of literal sets between prompt and answer (serves as a prior). Weights sum to 1 (e.g., w1=0.3, w2=0.2, w3=0.2, w4=0.2, w5=0.1). Higher S indicates a more coherent, consistent answer.
