## Nous Analysis

**Algorithm: Adaptive Bandit‑Guided Dual‑Process Reasoner (ABDG‑R)**  

The tool treats each candidate answer as an “antigen” that must be evaluated by an immune‑inspired repertoire of reasoning “antibodies.” Each antibody encodes a lightweight logical rule extracted from the prompt (e.g., “if X > Y then Z”). The repertoire is initialized with a set of naïve antibodies generated by regex‑based structural parsing (see §2).  

**Data structures**  
- `Antibody`: tuple `(condition_lambda, action_lambda, affinity, usage_count)`.  
  - `condition_lambda` : a Python lambda that returns True/False when applied to a parsed feature vector of the answer.  
  - `action_lambda` : returns a raw score contribution (e.g., +1 for satisfied conditional, –0.5 for violated negation).  
  - `affinity` : float in [0,1] representing current estimated quality (updated via clonal selection).  
  - `usage_count` : integer for bandit bookkeeping.  
- `Repertoire`: list of `Antibody` objects.  
- `FeatureVector`: numpy array of binary/int features extracted from the answer (negations, comparatives, numerics, causal flags, ordering relations).  

**Operations**  
1. **Clonal selection & affinity maturation** – For each answer, compute raw affinity as the dot product of antibody affinities with their condition matches:  
   `raw = Σ affinity_i * condition_i(fv)`.  
   Select the top‑k antibodies (e.g., k=5) proportional to raw affinity, clone them, and mutate their `action_lambda` by adding small Gaussian noise to numeric thresholds (simulating somatic hypermutation).  
2. **Dual‑process scoring** –  
   - *System 1* (fast): sum of unmutated naïve antibody contributions (baseline intuition).  
   - *System 2* (slow): sum of contributions from the mutated clonal set (deliberate refinement).  
   Final score = `α * System1 + (1-α) * System2`, where α decays over answers (starting 0.7 → 0.3) to shift from intuition to analysis as evidence accumulates.  
3. **Multi‑armed bandit update** – Each antibody is an arm. After scoring an answer, observe reward `r = normalized final score`. Update antibody affinity via Thompson‑sampling‑style Beta posterior:  
   `affinity = Beta(1+Σr_i·condition_i, 1+Σ(1−r_i)·condition_i).mean()`.  
   Increment `usage_count`. Arms with high uncertainty are explored more, balancing exploration of novel logical patterns against exploitation of high‑affinity rules.  

**Structural features parsed (regex‑based)**  
- Negations (`not`, `no`, `-`).  
- Comparatives (`greater than`, `<`, `>`, `≥`, `≤`).  
- Numerics (integers, floats, percentages).  
- Causal cues (`because`, `therefore`, `if … then`).  
- Ordering relations (`first`, `then`, `finally`, `before/after`).  
- Quantifiers (`all`, `some`, `none`).  

Each feature yields a binary or scalar entry in `FeatureVector`.
