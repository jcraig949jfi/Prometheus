**Algorithm: Sparse‑Measure Feedback Scorer (SMFS)**  
The tool represents each candidate answer as a high‑dimensional feature vector **x** ∈ ℝᴰ, where each dimension corresponds to a parsed structural element (see §2). A dictionary **Φ** ∈ ℝᴰˣᴷ (K ≪ D) is learned offline from a corpus of high‑quality reference answers using **measure‑theoretic weighting**: each column φₖ is weighted by the Lebesgue measure of the set of texts exhibiting that pattern under a σ‑algebra generated by linguistic primitives (e.g., clauses, quantifiers). This yields a measure‑biased sensing matrix that favors rare but informative patterns.

Given a new answer, we compute its measurement **y = Φᵀx** (compressed sensing step). Because **x** is expected to be sparse in the basis of meaningful linguistic patterns, we recover an estimate **x̂** by solving the convex program  

 min‖z‖₁ s.t. ‖Φᵀz − y‖₂ ≤ ε,  

using a standard Basis Pursuit solver (e.g., L1‑LS via numpy.linalg.lstsq with an L1 penalty via iterative soft‑thresholding). The residual **r = y − Φᵀx̂** quantifies how well the answer conforms to the learned sparse structure.

Feedback control enters by treating the residual as an error signal **eₜ = ‖r‖₂** that drives a discrete‑time PID controller adjusting a scalar confidence gain **gₜ**:  

 gₜ₊₁ = gₜ + Kₚeₜ + Kᵢ∑ₖ₌₀ᵗeₖ + Kᵈ(eₜ − eₜ₋₁).  

The final score is **s = gₜ·(1 − ‖r‖₂/‖y‖₂)**, clipped to [0,1]. High scores arise when the answer exhibits a sparse, measure‑weighted pattern set that matches the reference distribution, and the PID loop quickly damps residual error.
