## Nous Analysis

**Algorithm: Abductive‑Emergent Sensitivity Scorer (AESS)**  

1. **Data structures**  
   - *Clause graph*: Directed acyclic graph where each node is a parsed proposition (subject‑predicate‑object triple) extracted via regex patterns for negations, comparatives, conditionals, causal connectives (“because”, “leads to”), and numeric expressions. Edges represent logical relations (entailment, contradiction, conditional).  
   - *Explanation set*: List of candidate hypotheses generated by abductive step; each hypothesis is a sub‑graph that, when added, makes the observed premises jointly consistent.  
   - *Sensitivity matrix*: NumPy array S of shape (|H|, |P|) where H = hypotheses, P = perturbed premise copies; S[i,j] measures change in hypothesis i’s support score when premise j is replaced by a perturbed version (e.g., negated, numeric ±δ, or swapped comparator).  

2. **Operations**  
   - **Parsing**: Regex extracts propositions, tags modality (negation, possibility), and builds the clause graph.  
   - **Abductive generation**: For each missing link that would cause inconsistency (detected via a simple SAT‑like propagation using transitivity and modus ponens on the graph), generate a hypothesis node that fills the gap. Score each hypothesis by *explanatory virtues*: (a) coverage (number of premises explained), (b) simplicity (inverse of node count), (c) coherence (absence of internal contradictions).  
   - **Emergence aggregation**: Compute macro‑level support for each hypothesis as the weighted sum of its constituent micro‑proposition scores (weights = explanatory virtues). This yields a emergent score E[h].  
   - **Sensitivity analysis**: Create perturbed premise sets by applying atomic perturbations (negation flip, comparator reversal, numeric ±ε). For each hypothesis, recompute E[h] on each perturbed set; fill S. Compute robustness R[h] = 1 – (std(S[h,:]) / mean(|S[h,:]|+1e‑6)).  
   - **Final score**: Score(h) = E[h] × R[h]. Candidate answers are ranked by this product; ties broken by lexical specificity (longer hypothesis).  

3. **Parsed structural features**  
   - Negations (“not”, “no”), comparatives (“greater than”, “less than”), conditionals (“if … then …”), causal claims (“because”, “due to”), numeric values with units, ordering relations (“first”, “second”, “before”, “after”).  

4. **Novelty**

While abduction and sensitivity analysis appear separately in diagnostic systems, their integration with an emergence‑level aggregation step is novel for text‑based reasoning evaluation.
