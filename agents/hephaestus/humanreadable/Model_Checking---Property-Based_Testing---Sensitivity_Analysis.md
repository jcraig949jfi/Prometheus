**Algorithm**  
We build a finite‑state model of the premises and treat each candidate answer as a temporal‑logic property to be verified.  

1. **Parsing → propositional/numeric constraints**  
   - Extract atomic propositions \(p_i\) (e.g., “X > Y”, “Z caused W”) using regex patterns for negations, comparatives, conditionals, causal cues, and ordering words.  
   - Encode each proposition as a tuple \((\text{id}, \text{polarity}, \text{args})\); numeric arguments become entries in a NumPy array \(x\).  
   - Store constraints in two structures: a list \(C_{\text{bool}}\) of Boolean clauses and a matrix \(A\) · \(x \le b\) for linear inequalities (derived from comparatives).  

2. **State‑space construction (model checking)**  
   - Each variable (Boolean or discretized numeric) gets a bit‑width; a global state is an integer \(s\) whose bits encode the current truth assignment.  
   - The transition relation \(T\) is generated by applying all permissible actions (e.g., “if \(p\) then \(q\)”) as bit‑wise updates; this yields a directed graph \(G=(S,T)\).  
   - Perform BFS/DFS from the initial state(s) to enumerate reachable states \(S_{\text{reach}}\). During exploration, propagate unit clauses and apply transitivity on ordering constraints to prune infeasible states (constraint propagation).  

3. **Property‑based testing of the answer**  
   - Convert the candidate answer into a Linear Temporal Logic (LTL) formula \(\phi\) (e.g., “G (p → F q)”).  
   - Build a Büchi automaton \(B_{\phi}\) via a simple tableau (no external libraries).  
   - Compute the product automaton \(P = G \times B_{\phi}\). If \(P\) has no accepting cycle, \(\phi\) holds in all reachable states → score = 1.  

4. **Sensitivity‑driven shrinking (property‑based testing + sensitivity analysis)**  
   - If an accepting cycle exists, extract a counterexample path \(\pi = s_0,s_1,…,s_k\).  
   - For each numeric variable \(x_j\) on \(\pi\), perturb its value by \(\pm\epsilon\) (using NumPy) and re‑evaluate the path; record the minimal perturbation \(\delta_j\) that eliminates the accepting cycle.  
   - Apply a shrinking loop: repeatedly try to reduce \(\delta_j\) by half while the cycle persists, yielding a minimal failing perturbation \(\Delta = \max_j \delta_j\).  
   - Final score = \(\exp(-\lambda \Delta)\) with \(\lambda=1/\text{max\_possible\_perturb}\) (pre‑computed from variable ranges).
