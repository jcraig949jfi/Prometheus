## Nous Analysis

**Algorithm: Redundant Metamorphic Sensitivity Scorer (RMSS)**  

1. **Data structures**  
   - `tokens`: list of parsed atomic propositions extracted from the prompt and each candidate answer using regex patterns for negations (`not`, `never`), comparatives (`greater`, `less`, `more than`), conditionals (`if … then`, `unless`), causal verbs (`cause`, `lead to`, `result in`), and numeric expressions (integers, floats, percentages).  
   - `relation_matrix` ∈ {0,1}^{n×n}: binary adjacency where `relation_matrix[i,j]=1` iff token *i* logically implies token *j* (derived from extracted conditionals, comparatives, and causal chains).  
   - `codebook`: a (n, k) generator matrix **G** of a systematic linear block code (e.g., Hamming(7,4) or a short LDPC) built once from `n` (number of tokens).  
   - `perturb_set`: a small set of input perturbations generated by metamorphic relations (MRs):  
        * MR1 – duplicate a conjunctive clause,  
        * MR2 – swap the order of two independent clauses,  
        * MR3 – negate a single literal,  
        * MR4 – add/subtract a constant to every numeric token.  

2. **Operations**  
   - For each candidate answer, build its token vector **x** ∈ {0,1}^n (1 = token present).  
   - Encode **x** → **c** = **x**·**G** (mod 2) using only numpy’s dot and `%2`.  
   - Apply each MR to the original prompt, re‑tokenize, obtain perturbed vectors **x′**, encode to **c′**.  
   - Compute sensitivity: `s = mean_hamming_distance(c, c′)` over all MRs (numpy `bitwise_xor` + `sum`). Low `s` indicates the answer’s logical structure is stable under metamorphic changes → higher robustness.  
   - Simultaneously run constraint propagation on `relation_matrix` (Floyd‑Warshall transitive closure with numpy boolean `dot`/`or`) to detect violations (e.g., A→B, B→¬A). Each violation adds a penalty `p`.  
   - Final score: `score = α * (1 - s / s_max) - β * p`, where `α,β` are tuned weights (e.g., 0.7,0.3) and `s_max` is the maximum possible Hamming distance for the code length.

3. **Parsed structural features**  
   - Negations, comparatives, conditionals, causal claims, ordering relations, and numeric constants. These are the atoms that become bits in **x** and the edges in `relation_matrix`.

4. **Novelty**  
   - Error‑correcting codes provide a distance‑based robustness metric; metamorphic testing supplies a principled set of input perturbations without an oracle; sensitivity analysis quantifies how output (the codeword) changes under those perturbations. The combination—using a linear block code to embed logical token vectors, then measuring codeword stability under MR‑derived perturbations while propagating logical constraints—is not described in existing surveys of automated reasoning scorers, making it novel.
