**Algorithm – Critical‑Measure Compression Scorer (CMCS)**  

1. **Parsing & Graph Construction**  
   - Tokenize the prompt and each candidate answer with a regex‑based tokenizer that extracts:  
     * literals (numbers, entities)  
     * logical operators (¬, ∧, ∨, →) from cue words (“not”, “and”, “or”, “if … then”)  
     * comparative predicates (“>”, “<”, “=”, “more … than”)  
     * causal markers (“because”, “leads to”, “results in”)  
     * ordering relations (“before”, “after”, “first”, “last”)  
   - Build a directed labeled graph **G = (V, E)** where each vertex *v* ∈ V is a proposition (a literal possibly wrapped with a negation flag) and each edge *e* = (u → v, ℓ) carries a label ℓ ∈ {COND, COMP, CAUS, ORD}.  
   - Store **G** as two numpy arrays: an adjacency matrix **A** (|V|×|V|, dtype=int8) where A[i,j]=k encodes the label *k* (0 = no edge, 1‑4 for the four relation types) and a feature vector **f** (|V|, dtype=bool) indicating negation.

2. **Measure‑Theoretic Weighting**  
   - Define a σ‑algebra on the power set of V generated by the edge‑label partitions.  
   - Assign a base measure μ₀(S) = |S|/|V| (Lebesgue‑like uniform measure).  
   - Propagate a **criticality score** c(v) for each vertex using a sandpile rule:  
     * Initialize c(v) = μ₀({v}).  
     * For each vertex, if c(v) > θ (θ = 0.2, a critical threshold), topple: distribute excess equally to its outgoing neighbors (c(u) ← c(u) + excess/outdeg(v)) and set c(v) ← θ.  
     * Iterate until no vertex exceeds θ (O(|V|·|E|) worst‑case, but converges quickly in practice).  
   - The final vertex measures **μ(v) = c(v)** form a probability‑like distribution over propositions.

3. **Similarity via Normalized Compression Distance**  
   - Serialize each graph **G** into a canonical string: list vertices sorted by identifier, each entry “id:neg|label‑list”, edges appended as “id→id:label”.  
   - Apply Python’s `zlib.compress` (available in stdlib) to obtain byte lengths L(G).  
   - For a candidate answer **Gc** and a reference answer **Gr** (derived from the prompt’s expected logical skeleton), compute  
     NCD(Gc,Gr) = (L(C(Gc‖Gr)) – min(L(Gc),L(Gr))) / max(L(Gc),L(Gr)),  
     where C denotes concatenation and compression.  
   - The final score is **S = 1 – NCD**, weighted by the measure entropy H = – Σ μ(v) log μ(v): **Score = S · (1 + H)**. Higher scores indicate answers whose logical structure is both compress‑similar to the reference and critically balanced across propositions.
