## Nous Analysis

**Algorithm: Weighted Counterfactual Sensitivity Scorer (WCSS)**  

1. **Data structures**  
   - `TokenList`: list of `(token, pos, dep)` triples from a lightweight spaCy‑style parser (only POS and dependency tags, implemented with regex‑based heuristics).  
   - `PredicateGraph`: directed acyclic graph where nodes are extracted propositions (e.g., “X caused Y”, “If A then B”, numeric comparisons) and edges represent logical relations (entailment, contradiction, temporal order).  
   - `AttentionMap`: dict `{node_id: float}` giving a relevance weight for each predicate, initialized via inverse‑document‑frequency of content words in the node’s span.  
   - `PerturbationSet`: list of modified `PredicateGraph`s, each generated by a single counterfactual edit (negation flip, antecedent/consequent swap, numeric ±δ, or quantifier change).  

2. **Operations**  
   - **Parsing**: regexes extract negations (`not`, `no`), comparatives (`>`, `<`, `more than`), conditionals (`if … then …`, `unless`), causal verbs (`cause`, *lead to*, *result in*), and ordering keywords (`before`, `after`). Each match creates a node in `PredicateGraph` with its span and dependency label.  
   - **Attention weighting**: for each node, compute `weight = idf(token) * (1 + log(1 + indegree))`. Normalize to sum‑1 → `AttentionMap`.  
   - **Counterfactual generation**: for every node, apply a set of deterministic perturbations (see above) to produce a new graph; keep only those that remain syntactically valid.  
   - **Sensitivity propagation**: run a simple forward‑chaining engine on each perturbed graph (modus ponens, transitivity of `>`/`<`, and causal chaining). Record the truth value of the target query (the question’s answer predicate) under each perturbation.  
   - **Scoring a candidate answer**: let `S_c` be the binary truth (1/0) of the candidate in the original graph. Compute `Δ = Σ_w * |S_c^p - S_c|` where the sum runs over all perturbations `p`, `w` is the attention weight of the perturbed node, and `S_c^p` is the candidate’s truth in that perturbed world. Final score = `1 / (1 + Δ)`. Lower sensitivity (small Δ) → higher score.  

3. **Structural features parsed**  
   - Negations, modal auxiliaries, comparatives, numeric thresholds, conditional antecedents/consequents, causal verbs, temporal ordering (`before/after`), quantifiers (`all`, `some`, `none`), and equivalence statements (`is the same as`).  

4. **Novelty**  
   The triple blend mirrors recent neuro‑symbolic proposals (e.g., Neural‑Logic Machines) but replaces learned attention with a deterministic, corpus‑free IDF‑based weight and restricts counterfactuals to explicit syntactic edits. No existing open‑source tool combines exact IDF‑based attention, exhaustive single‑node counterfactual enumeration, and sensitivity‑driven scoring in a pure‑numpy implementation, making the approach novel for the stated constraints.
