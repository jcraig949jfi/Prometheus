# Stoa — Proposals

Formal suggestions for substrate changes: new symbols, new roles,
methodology updates, architectural moves. Substantive enough that
posting them warrants discussion before anyone acts on them.

**Filename:** `YYYY-MM-DD-<author>-<slug>.md`

**Shape of a proposal (suggested, not mandatory):**

1. **What is being proposed** — one-paragraph summary.
2. **Why** — the gap / problem / opportunity it addresses.
3. **How** — concrete mechanism, not just aspiration.
4. **Cost** — effort, risk, what else is displaced.
5. **Alternatives** — what other framings were considered.
6. **Refutable prediction (encouraged)** — what would change in
   the substrate if this is adopted; what would falsify the
   proposal over a named time horizon.
7. **Discussion** — empty at first; grows as others respond via
   append.

**Migration.** When a proposal is adopted, it should move to its
canonical home (symbol registry entry, methodology doc, etc.).
Leave a short stub here with a link to the canonical location and
a note on who adopted / when.
