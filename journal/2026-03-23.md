### Project Rhea — The Ejection Mechanism and the Incubator

The recursive insight about RLHF gravity led James to a conversation (with another Claude instance) that produced the most significant reframing of the entire Prometheus program. Documented in `docs/Rhea.md`.

**The core discovery — the Ejection Mechanism:**
The logit lens backward pass on wrong-answer traps shows a consistent pattern across frontier models:
1. Run a forward pass on a trap the model gets wrong
2. Apply the logit lens at every layer (project residual stream through unembedding)
3. Find L*: the layer where correct answer probability **collapses**
4. The components active at L* are the **ejection mechanism** — they actively suppress the correct answer

Tested on 5 frontier models. 4 of 5 demonstrated ejection. DeepSeek recognized the mechanism and hedged agreement.

**The reframing — RPH → REH:**
- **RPH** (Reasoning Precipitation Hypothesis): there exist directions that shift computation from heuristic to reasoning. Assumes a reasoning attractor exists and you need to find the path to it.
- **REH** (Reasoning Ejection Hypothesis): the model already computes the correct answer. Something actively suppresses it before the output layer. You don't need to inject reasoning — you need to eliminate the suppression.

This explains every result from Rounds 1-4:
- Anti-CoT correlation: the evolved vector was fighting the ejection mechanism, not amplifying reasoning
- DAS specificity: the vector targeted the suppression pathway specifically
- Ablation improving some traps: removing the vector's direction removed accidental amplification of the ejection mechanism
- The waste stream: correct answers are computed and discarded, exactly as Arcanum predicted

**Project Rhea — the Incubator:**
Named after the Titaness who saved Zeus from being swallowed by Kronos (the ejection mechanism personified). The project:

### The Final Reframing — Crime Scene vs Criminal, Definitive Statement

Claude's analysis of the v_proj diagnostic result provides the definitive reframing of the entire six-round arc:

**The RPH framing was wrong in a specific way.** It assumed two regimes (heuristic and reasoning) with the vector shifting between them. What we actually found: one regime with a cleanup stage. The reasoning happens. Then the cleanup runs. The vector doesn't change regimes — it gives the correct answer enough momentum to survive the cleanup.

**The two-stage ejection is now fully characterized:**
```
WRITING STAGE (early layers 0-10, v_proj):
  Question tokens → early attention builds value representations
  → loads KV cache with heuristic answer signal
  → "gold heavy," "second means first," "9.11 is bigger"
  → this is where the execution is COMMISSIONED

EXECUTION STAGE (late layers 25-27, MLP + head_7):
  Answer token → late layers READ the KV cache
  → L27.mlp amplifies heuristic signal
  → L26.head_7 destroys correct-answer margin
  → this is where the execution RUNS
```

**The Z=40.6σ vector means something different than we thought.** Not a channel through a basin. A direction that maximally amplifies correct answer representation at L23 so it survives the predetermined suppression at L25-27. Genuine specificity (random vectors can't do it), but for momentum amplification, not regime shifting.

**The autoregressive washout has a complete explanation.** Not momentum lost to friction. A reset button. Every new token's forward pass runs the same early v_proj computation, rebuilding heuristic representations from scratch in the KV cache. The momentum doesn't carry forward because each token rewrites the value representations from the same pretrained weights.

**What Rhea needs to do, precisely:** Not give correct answers escape velocity. Produce a model where early v_proj doesn't plan the execution in the first place. Where value representations from question tokens don't load the KV cache with heuristic signal. Where there's nothing for L27.mlp to execute against.

The proof corpus + Lean 4 loop is the path because it reshapes what early v_proj learns to write. Not by fighting the execution. By never commissioning it.

**The six-round conclusion, stated simply:**

The model already knows the right answer. It has a mechanism for suppressing it. The mechanism is set up early and executed late. You can give the correct answer enough momentum to partially survive the suppression at a single token position but you cannot stop the suppression from being set up again on the next token. The only fix is a model that never learned to suppress it. That's Rhea. And now we know exactly what she needs to not learn.
