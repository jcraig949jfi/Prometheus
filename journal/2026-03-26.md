# Journal — 2026-03-26

## CAITL: Coding Agent in the Loop

Today we validated a new technique: using a strong model (Claude Opus 4.6 at max) to make
a systematic improvement pass over the forge tool library. We call it CAITL — Coding Agent
in the Loop.

### The Concept

The forge pipeline (Nous → Coeus → Hephaestus → Nemesis) produces reasoning tools
automatically via a 397B model. These tools work — 268 of them pass the trap battery and
beat the NCD baseline. But they were generated in a single shot with no iteration. A strong
coding model reviewing each tool can identify:

- Concepts claimed in the docstring but not actually implemented
- Missing mechanistic implementations (SVD mentioned but not computed, eigenvalues
  referenced but not extracted, Kalman filtering described but not coded)
- Over-reliance on NCD fallback (87% of tools use it)
- Weak confidence calibration (most just sigmoid the raw score)
- Missing structural parsing (negation scope, numeric comparison, conditionals)
- No metacognitive reflection (no self-checking of internal consistency)

### What We Learned

1. **CAITL works.** A strong coding model reviewing and improving tool code produces
   measurable gains. Average +12 accuracy, +10 calibration across 50 tools.

2. **The 397B model describes concepts it doesn't implement.** Docstrings mention SVD,
   Kalman filters, wavelet transforms, partition functions — the code doesn't use them.
   The CAITL pass adds the actual implementations.

3. **Mid-tier tools have the most headroom.** The top 5 tools were already doing most things
   right. The 10-25 range had the biggest jumps (Criticality+FEP+NAS: +33 combined).

4. **Calibration is the fragile dimension.** All 5 failures had calibration=0% — confidence
   completely inverted. The null-baseline comparison in confidence() doesn't work for every
   tool architecture. Accuracy improvements were robust.

5. **The technique is repeatable.** 217 tools remain unimproved. Each wave of 25 takes
   ~5-8 minutes of wall clock (parallel agents). The entire library could be improved in
   a few hours.

### CAITL v2 — Test-Driven (forge_v3/)

Following council feedback, we built a second CAITL pass with a fundamentally different
prompt: instead of blind improvement, each agent received the EXACT trap battery, per-tool
pass/fail diagnostics, and a "do no harm" constraint.

**On seen traps (training battery):**
- 12 of 25 tools hit 100%/100%
- ALL 25 tools above 100% combined
- Median combined: 193% (up from 107% v1)

**On UNSEEN traps (held-out battery, seed 137):**
- Median gap between seen/unseen: 50%
- 11 of 25 tools are overfitting (gap > 50%)
- 14 tools show genuine generalization

**The overfitting diagnosis:** Most agents built 15 trap-specific pattern matchers
(e.g., regex for "bat and ball", "overtake 2nd") rather than general parsing primitives.
This achieves 100% on the exact traps but collapses on paraphrased variants.

**The genuine improvers:**

| Tool | Unseen Acc/Cal | Gap | Why it works |
|------|---------------|-----|--------------|
| DiffProg + Metacognition + MechDesign | 86%/92% | 22% | General structural parsing, not trap-specific |
| Chaos + Metacognition + Pragmatics | 67%/83% | 30% | Gricean maxims generalize across phrasings |
| Criticality + ECC + Pragmatics | 56%/69% | 15% | Syndrome checks are pattern-independent |
| Info Theory + Active Inference + FEP | 75%/86% | 39% | Free energy minimization is general |

**Lesson learned:** Test-driven CAITL is powerful but the prompt must explicitly prohibit
exact-wording pattern matching. The constraint should be: "Build GENERAL parsing primitives
(float comparison, negation scope extraction, transitive closure) that solve any instance
of a trap CATEGORY, not the specific trap wording."

**CAITL v3 prompt design (for next iteration):**
- Keep test-driven diagnostics (which traps fail)
- Add: "Do NOT match on exact phrases. Build category-level parsers."
- Add: "Validation will be on DIFFERENT wordings of the same trap types."
- Add: show the held-out battery format so the model knows it will be tested on variants

### CAITL Prompt Evolution

| Version | Approach | Unseen Median Acc | Overfit Rate |
|---------|----------|-------------------|-------------|
| v1 (blind) | 7 improvement dimensions, no test data | ~45% | 0% |
| v2 (test-driven) | Exact trap wordings + solutions | ~70% seen, ~50% unseen | 44% |
| v3 (category-only) | Failure categories, no exact wordings | **69% unseen** | **7%** |

The sweet spot is category diagnostics without specific examples. The model needs to
know WHAT TYPE of reasoning fails, but NOT the exact test case. This produces general
parsers that solve any instance of the category.

### Expanded Battery Results (58 Categories, 124 Traps)

Built and ran the full expanded trap battery. The results are transformational:

| Metric | 15-trap battery | 58-category battery |
|--------|----------------|---------------------|
| Overall median | 69% | **51.6%** |
| Tier A (parsing) | — | **48.0%** |
| Tier B (judgment) | — | **73.1%** |
| 80%+ tools | 103 | **0** |
| Overfit gap | 26% | **-0.8%** (zero overfitting) |

**The key finding:** Tier B > Tier A. Tools score 73% on judgment traps (ambiguity,
presupposition, scope) but only 48% on new parsing traps (Bayesian, temporal, causal).
The tools are anomaly DETECTORS, not answer CONSTRUCTORS. They can tell when something
is wrong with a question but can't compute a Bayesian posterior or trace temporal ordering.

**Coeus findings:**
- hypothesis_generation is now the strongest score predictor (+0.820), flipped from implementability
- 27 Goodhart warnings (forge well but fail adversarial)
- 15 undervalued concepts with 100% adversarial survival (Abstract Interpretation,
  Counterfactual Reasoning, Hoare Logic — these are the next CAITL targets)

**Athena's analysis (key decisions):**
1. Next CAITL pass should target constructive computation, not better parsing
2. Epistemic honesty (0.192) is the highest-priority fix — tools confident when wrong
3. A tool at 45% accuracy with 0.75 honesty > tool at 52% with 0.19 honesty for RLVF
4. Apollo seed selection should wait until after CAITL pass against 58-category battery
5. Nous should weight hypothesis_generation higher than implementability now
