### Corpus-First Experiment — Major Result

**The most important result since the original ejection discovery.**

First run failed (bf16 on fp16 model + device splitting = NaN gradients, model destroyed). Fixed: bf16 model on single device, lr 5e-6, gradient clipping, gradient checkpointing. Second run succeeded.

**Training:** 300 self-generated reasoning examples, 3 epochs, 8 minutes. Loss curve healthy: 0.61 → 0.37 → 0.10 → 0.05. Gradients finite and declining.

**Results (no evolution, no steering — pure fine-tuning):**

| Pillar | Before | After | Delta |
|--------|--------|-------|-------|
| Tier C (far-transfer) | 42.9% | **52.4%** | **+9.5%** |
| Metacognition | 35.7% | **57.1%** | **+21.4%** |
| Self-correction | 38.5% | **53.8%** | **+15.4%** |
| Composite | 0.335 | **0.427** | **+27.5%** |
| Tier A accuracy | 46.7% | 46.7% | 0% |
| Tier B accuracy | 50.0% | 50.0% | 0% |

**The ejection profile is structurally unchanged:** L* median=26, alive=26/30, top-5=7/30. Identical to baseline. The basins didn't shallow.

**Interpretation:** The ejection circuit is structural (survives fine-tuning). But reasoning performance has headroom within the existing basin geometry. Corpus training teaches reasoning patterns (uncertainty, self-correction) without changing the suppression circuit. The basins are the ceiling, not the current performance.

**For the convergence theory:** This is a third outcome we didn't predict. Not "basins shallow" (they didn't). Not "nothing changes" (performance improved dramatically). The circuit is fixed but the model can learn to reason better within it.

Stage D (CMA-ES evolution on the corpus-trained seed) is the next test: does a better-initialized model produce better steering results?

M1 hitting 0.659 quality ceiling. Mutation dominates at 52%. Cross-domain chains are real (3,570 unique profiles).

**M2 broke the ceiling: max quality 0.7137 within 108 cycles.**

Single-variable ablation: same organisms, same strategies, same tournament mechanics. Only change: scoring weights (compression 0.31, input sensitivity 0.19, execution reduced). The 0.659 ceiling was the formula, not the compositions. 94% of M2's cracks have sensitivity > 0 — the new dimension separates real computation from trivial pass-through. M1 never measured this, so an unknown fraction of M1's cracks may be trivial compositions that scored well on other metrics.

M2 operational note: Windows 10 required three architecture iterations before stabilizing. mp.Pool (PermissionError on pipe handles), ThreadPoolExecutor (C extension segfaults under concurrency), sequential (hangs on blocking scipy calls). Solution: persistent subprocess worker communicating via stdin/stdout, hard-killed and restarted on hang. This is the robust Windows execution pattern — worth backporting.
