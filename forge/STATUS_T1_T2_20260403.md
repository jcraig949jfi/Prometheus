## Recommended Changes for T2

### 1. Threshold recalibration -- DONE

Applied 2026-04-03. `pass_threshold` 0.50 to 0.40, `max_seed_drop` 0.15 to 0.20. Recovered 5 tools from the existing population. See thresholds table above.

### 2. Category-cluster evaluation -- PROPOSED (P1)

**Problem:** The current 12-category cross-eval is unrealistic for specialist tools. A single-domain specialist's theoretical maximum is ~37.5% (it can only score on the categories it was built for, which are a fraction of the 12). This creates a structural ceiling that punishes specialization.

**Proposal:** Replace the 12-category cross-eval with reasoning-family clusters. Test generalization *within* a domain (e.g., simpson_paradox + causal_counterfactual + causal_confounding_hard), not across all domains. A tool that deeply solves its family should be able to pass without needing to also solve scheduling and liar detection.

**Expected impact:** Unlocks specialist tools that are currently walled out by the cross-eval ceiling.

### 3. Re-evaluate historical candidates -- P1

75 candidates evaluated before today's threshold change. Re-run verdicts against new thresholds (no re-evaluation needed, just re-apply pass criteria to existing scores). Near-misses like `liar_detection_012` (38.3%, seed_drop 8.3pp) are close and may pass under cluster evaluation.

### 4. Builder prompt: enforce load-bearing primitives -- P2

Current tools import T1 primitives decoratively (ablation delta = 0.0). The builder prompt needs to enforce that primitives are load-bearing:
- Require that ablating any primitive changes at least 20% of outputs (`min_ablation_impact` already exists in `thresholds.py` but is not driving generation)
- Add explicit ablation self-test to the builder's output template
- Reference: `forge/builder_prompt.py`

### 5. Gem-forging pipeline: scale what works -- P1

The gem-forging pattern is proven and high-ROI:
- Raw tool at ~42% -> gem-forged to ~97% via Claude Code refinement
- Key transformation: narrow extraction -> universal extraction + multi-strategy routing
- Immediate targets: `perspective_shift_005` (44.2%, needs seed stabilization), `perspective_shift_019` (39.2%, needs +0.8pp), `liar_detection_012` (38.3%, needs +1.7pp)
- Each gem-forge takes ~15 minutes of Claude Code time and yields a 2x+ score improvement
